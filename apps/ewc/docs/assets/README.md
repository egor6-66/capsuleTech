Сюда складываются скриншоты и диаграммы для страниц документации. Пока реальных изображений нет — в тексте стоят **текстовые плейсхолдеры**, размечающие, какая картинка куда встанет.

---

## Конвенция плейсхолдера

В странице плейсхолдер — это блок-цитата сразу после вводного абзаца:

```markdown
> 🖼️ **Скриншот:** _Подпись курсивом — что на картинке._ `../assets/имя-файла.png`
```

Почему именно так:

| Решение | Причина |
|---|---|
| Не `![](...)`, а текст + путь в \`코де\` | «Битой картинки» нет, пока файла нет — страница выглядит цельно |
| Путь относительный (`../assets/...`) | Работает и на GitHub, и в Obsidian, и при выносе в отдельный репозиторий |
| Подпись курсивом | Сразу понятно, что снимать, ещё до появления файла |

Когда появляется реальный файл — плейсхолдер меняется на обычную картинку:

```markdown
![Подпись](../assets/имя-файла.png)
```

---

## Правила именования

| Правило | Пример |
|---|---|
| `kebab-case`, по теме страницы | `map-table-sync.png` |
| Без пробелов и кириллицы в имени файла | ✅ `auth-login.png` · ❌ `вход.png` |
| Формат — PNG (скриншоты), SVG (диаграммы) | `incident-card.png` |
| Ширина ~1280–1920px, без лишних полей | — |
