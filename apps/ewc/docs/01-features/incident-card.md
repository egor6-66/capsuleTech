## Что это и зачем

Карточка открывается двумя путями:

| 👤 Вы сделали | | ⚡ Что происходит |
|---|:---:|---|
| **Двойной клик** по строке или маркеру | → | 📄 открывается детальная карточка `/workspace/cards/:id` |
| Выбрали инцидент → нажали **«Открыть карточку»** в сайдбаре | → | 📄 та же детальная карточка |

Форма из 5 секций:

| Секция | Что содержит |
|---|---|
| **Информация** | № карточки, даты/время, источник, тип обращения и происшествия |
| **Данные заявителя** | ФИО, телефон, адрес, аудиозапись |
| **Описание происшествия** | объект, координаты, флаги (ЧС, авария ЖКХ…), содержание |
| **Организация** | управляющая компания, ответственная орг. |
| **Служебная информация** | статус, даты установки, отчёт о выполнении |

## Как работает под капотом

🛠️ _Для разработчиков._

### Форма — это данные, а не вёрстка

Карточка описана как `ISchema` (дерево `IEditorNode`) в [`rendererSchemes/incidentCard.ts`](../../src/rendererSchemes/incidentCard.ts) и рисуется пакетом `@capsuletech/web-renderer`:

```ts
// pages/workspace/cards/[id]/index.tsx
const registry = { ui: Ui } as unknown as Registry;
<Renderer schema={incidentCardSchema} registry={registry} mode="static" />
```

- **`schema`** — JSON-дерево нод (`ui.Field`, `ui.Input`, `ui.Layout.Grid`…), собранное декларативно из спецификаций секций.
- **`registry`** — собирается из проксированного `Ui`; рендерер резолвит dot-path'ы (`ui.Field.Label`, `ui.Input`…) прямо по нему.
- **`mode="static"`** — read-only показ дефолта.

> [!TIP]
> **Зачем такая сложность?** Раз форма — это данные, её можно будет редактировать визуально через `@capsuletech/web-ui-creator` (операции `addNode`/`moveNode` над деревом). Это закладка под будущий конструктор форм — см. [Бэклог платформы](../03-roadmap/backlog/platform.md).

### Мастер-деталь

Список и деталь живут на соседних маршрутах ([`pages/workspace/cards/index.tsx`](../../src/pages/workspace/cards/index.tsx)):

| Маршрут | Что |
|---|---|
| `/workspace/cards` | список карточек (та же таблица инцидентов) |
| `/workspace/cards/:id` | детальная карточка в `<Outlet/>` |

Переключатель реактивен: `useRouter().current()` читается через Solid-сигнал, `<Show>` перерисовывается на каждую навигацию.

## Заглушки

⚠️ В `@capsuletech/web-ui` пока нет части примитивов — они заменены ближайшими ([`rendererSchemes/incidentCard.ts`](../../src/rendererSchemes/incidentCard.ts)):

| Нужен контрол | Заглушка | Последствие |
|---|---|---|
| `select` (выпадающий список) | `ui.Input` readonly + hint «— выберите —» | Нельзя выбрать значение |
| `textarea` (многострочное) | `ui.Input` увеличенной высоты | Один ряд вместо нескольких |
| `checkbox` | `ui.Toggle` (switch) | Визуально тумблер, не галочка |
| `audio` (плеер записи) | `ui.Typography` (текстовый плейсхолдер) | Нет проигрывания |
| Иконки (телефон/карта/скачать), маски ввода (`+7`) | — | Опущены |

Плюс ограничения режима:

| ⚠️ | Что |
|---|---|
| 🔒 | Карточка **read-only** (`mode="static"`) — редактирование не сохраняется |
| 🧪 | Значения — мок одного инцидента; реальные данные по `id` из Feature придут позже |

Что нужно, чтобы закрыть → [Технический долг](../03-roadmap/tech-debt.md) и [Бэклог платформы](../03-roadmap/backlog/platform.md).
