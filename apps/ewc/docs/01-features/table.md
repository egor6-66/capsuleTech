> 🏠 [Хаб документации](../README.md) › ✅ [Фичи](README.md) › **Таблица инцидентов**

> **Аудитория:** 👤 Юзер · 🛠️ Разработчик · 📊 Менеджер

Таблица показывает все карточки происшествий списком: ID, заявитель, телефон, координаты, описание, время. Клик по строке выбирает инцидент — карта и сайдбар реагируют.

## Что это и зачем

Колонки таблицы:

| Колонка | Что показывает |
|---|---|
| **ID** | Номер карточки (`incident-0001`) |
| **Заявитель** | Имя заявителя |
| **Телефон** | Контактный номер |
| **Координаты** | Широта, долгота места происшествия |
| **Описание** | Что случилось |
| **Создано** | Дата и время регистрации (формат `ru-RU`) |

Что умеет:

| 👤 Вы сделали | | ⚡ Что происходит |
|---|:---:|---|
| Клик по **заголовку** колонки | → | 🔼 список сортируется по этой колонке |
| **Одиночный клик** по строке | → | ✅ инцидент выбран — строка подсвечена, карта и сайдбар реагируют |
| **Двойной клик** по строке | → | 📄 открывается детальная карточка происшествия |

## Как работает под капотом

🛠️ _Для разработчиков. Юзерам — сразу к [Шероховатостям](#шероховатости) или [How-to](../02-how-to/README.md)._

### Shape + Widget: разделение «что рисовать» и «чем кормить»

Таблица собрана из двух слоёв:

- **Shape** [`shapes/incidentsTable.tsx`](../../src/shapes/incidentsTable.tsx) — отвечает за **презентацию**: схема (`z.array(Entities.Incident.schema)`), `columns`, `sorting`, шаблон `ui.DataTable`. Не знает про live-данные.
- **Widget** [`widgets/tables/incidents.tsx`](../../src/widgets/tables/incidents.tsx) — подаёт **live-данные** из стора и навешивает поведение строк.

```ts
// widgets/tables/incidents.tsx
<Shapes.IncidentsTable
  data={data()?.items ?? []}
  itemMeta={() => ({ tags: ['incident', 'table'] })}   // ← каждая строка несёт теги
  itemPayload={(row) => ({ id: row.id })}              //   + payload с id
  isRowActive={(row) => row.id === data()?.selected?.id} // подсветка выбранной
  scrollToId={/* см. синхронизацию */}
/>
```

### Клик по строке — теги, а не колбэк

Строка помечена `tags: ['incident', 'table']` + `payload: { id }`. Клик перехватывает `web-core`, а универсальный [`Features.Incidents.onClick`](../../src/features/incidents.ts) роутит по тегам → кладёт выбранный инцидент в стор. Подробно — [🔗 Синхронизация](map-table-sync.md).

### Лоадер

Пока `store.loading === true`, Widget рисует table-скелетон (2-й колбэк фабрики):

```ts
(Ui) => <Ui.Skeleton variant="table" rows={100} />
```

Фича знает только про логический сигнал загрузки; **как** выглядит скелетон — решает Widget.

### Данные

Сейчас — 200 сидированных моков ([`Entities.Incident.mock`](../../src/entities/incident.tsx)), приходят через [`api.incidents.list()`](../../src/endpoints/incidents.ts) с задержкой 200 мс (виден loading-state). В prod-сборке моки вырезаются tree-shaking'ом, данные придут с реального бэкенда — [Phase 2](../03-roadmap/README.md).
