## Виртуальный скролл таблицы

> **Где:** `@capsuletech/web-ui` DataTable, режим `infinite` (virtual) · **Зона:** 🏗️ фреймворк (`owner-web-ui`)

### Симптом

После **холодной** перезагрузки (~раз в несколько раз, чаще под нагрузкой CPU) виртуальная таблица рендерит только spacer-`<tr>` без строк — тело пустое, хотя данные загружены. Уход на другую страницу и обратно (тёплый layout) — лечит.

### Корень

Timing + memoization race в `@tanstack/virtual-core` 3.14.0: `scrollRect` ставится только через `ResizeObserver`, а `maybeNotify` мемоизирован и может пропустить notify, когда scroll-элемент получает реальную высоту на фрейм позже. `measure()` не обновляет `scrollRect`.

### Обход (применён)

EWC-таблица переведена на **`plain`-режим** — non-virtual бэкенд `createInfiniteScroll`, который рендерит все подгруженные строки без виртуалайзера, поэтому quirk'а нет:

```ts
// shapes/incidentsTable.tsx
infinite: { itemHeight: 40, mode: 'plain' }
```

> [!WARNING]
> **Не возвращай `mode: 'virtual'`**, пока фреймворк не починит баг. Проверено: флаг `useAnimationFrameWithResizeObserver: true` **не помогает** (корень в initial rect=0 на mount). Flex-deferred высота content-wrapper'а усугубляет quirk до 100% — спасает overlay-стрип.

### Правильный фикс (будущее, зона фреймворка)

Keyed-remount виртуалайзера по приходу реальной высоты **или** апгрейд `@tanstack/solid-virtual` на исправленную версию. Верификация — **только в реальном браузере** (jsdom виртуалайзер не измеряет).

### Что это значит для EWC

На текущих объёмах (моки — 200 строк) `plain` работает отлично. Эффект «менее эффективно на больших списках» проявится только на реальных данных в тысячи строк — к тому моменту фреймворк-фикс закроет вопрос.

## Анимация перехода между страницами

> **Где:** [`pages/workspace/index.tsx`](../../src/pages/workspace/index.tsx) · **Зона:** 🏗️ нужна своя `Ui.Animate`-обёртка

### Симптом

Переходы между экранами workspace происходят **без анимации** — резкая смена контента.

### Что пробовали

Локальный `FadeIn` через `<For each={[pathname]}>` + opacity-сигнал — **откачено**, не тот подход. Нужна анимация через проектную либу `Ui.Animate` (поверх `solid-motionone`).

### Корень незавершённости

`<Animate keyed={pathname}>` не пере-mount'ит Motion на смену `keyed` — тот же DOM-нод остаётся, opacity не дрожит. Вероятно, lazy-wrapping `Ui.Animate` через `createLazy()` + Presence `resolveFirst` в `solid-motionone` не находит swap. Корень до конца не выяснен.

### Статус

Отложено до возврата к задаче (см. [Бэклог UX](backlog/ux.md#анимация-перехода-между-страницами)). Когда вернёмся — копать от поведения Presence/`resolveFirst`.
