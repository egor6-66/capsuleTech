# 🏗️ Бэклог — Платформа

> 🏠 [Хаб документации](../../README.md) › 🗺️ [Дорожная карта](../README.md) › 💡 [Бэклог](README.md) › **Платформа**

> **Аудитория:** 🛠️ Разработчик · 📊 Менеджер
> **Статус:** 📋 Идеи с оценками

Фундамент под боевой запуск: реальные данные, авторизация, редактирование, упаковка. Здесь живут самые «тяжёлые» пункты — большинство помечено 🔗 (зависит от бэкенда) и определяет [Phase 2](../README.md#этапы).

---

## Идеи

| 💡 Фича | Оценка | Что даёт | Зависимости |
|---|:---:|---|---|
| **Реальный API вместо моков** | 🟡 M (2–3) 🔗 | `api.incidents.list()` ходит в настоящий бэкенд | готовый эндпоинт |
| **Реальная авторизация + route guard** | 🟡 M (2–3) 🔗 | JWT + refresh, `/workspace` закрыт без входа | auth-эндпоинт |
| **Редактируемая карточка** | 🟡 M (3–5) | Карточка из read-only → controlled + сохранение | `web-renderer` controlled-режим |
| **Конструктор форм** | 🔴 L (5–10) | Визуальная сборка карточек без кода | `@capsuletech/web-ui-creator` |
| **Недостающие примитивы** | 🏗️ (3–5) | `select`, `textarea`, `checkbox`, `audio` в UI-ките | зона `owner-web-ui` |
| **Персистентность / live-обновления** | 🔴 L (5+) 🔗 | Инциденты обновляются в реальном времени (WS/SSE) | бэкенд push-канал |
| **Вынос в отдельный репозиторий** | 🟢 S (1–2) | EWC живёт как самостоятельный проект на Verdaccio | `@capsuletech/*` опубликованы |
| **Desktop (Tauri)** | 🟢 S (1–2) | EWC как нативное настольное приложение | `@capsuletech/desktop` (ADR 017) |

**Сумма app-части (без 🏗️ и открытых 🔴):** ~14–25 дней.

---

## Заметки по реализации

> [!NOTE]
> **Моки → API — это одна строка в каждом эндпоинте.** Слой [`endpoints/`](../../00-overview/architecture.md) уже изолирует данные через `@capsuletech/web-query`. Сейчас `preRequest` отдаёт мок; в проде убираем `preRequest` — `defineEndpoint` ходит в сеть. Поэтому 🟡 M, а не 🔴 L — архитектура к этому готова. Подробнее: [Технический долг](../tech-debt.md#временные-решения).

> [!IMPORTANT]
> **Авторизация и route guard — пара.** Сейчас `/workspace` открыт без входа, токен — мок в `localStorage` ([долг](../tech-debt.md#временные-решения)). Реальная авторизация без guard'а бессмысленна: делать вместе.

> [!TIP]
> **Редактируемая карточка переиспользует существующую схему.** Карточка уже рендерится из [`rendererSchemes/incidentCard.ts`](../../01-features/incident-card.md) через `web-renderer` в режиме `static`. Перевод в `controlled` + проброс сохранения — это не переписывание, а смена режима рендерера (+ закрытие [каста `Registry`](../tech-debt.md#типизация)).

> [!NOTE]
> **Конструктор форм — самый дорогой пункт (🔴 L).** Это полноценный design-time UI поверх [`@capsuletech/web-ui-creator`](../../00-overview/architecture.md): manifests + state-операции над JSON-деревом + инспектор. Окупается, только если карточек много и их меняют без разработчика.
