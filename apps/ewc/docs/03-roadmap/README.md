## Этапы

| Этап | Что входит | Статус |
|---|---|:---:|
| **Phase 0 — Demo** | UI трёх виджетов, синхронизация, DnD/ресайз, карточка-форма, моки | 🟢 в основном готово |
| **Phase 1 — Стабилизация** | починить виртуальный скролл (framework), убрать typing-касты, анимация переходов, защита маршрутов | 🟡 следующий |
| **Phase 2 — Реальные данные** | подключить API вместо моков, реальная авторизация, редактируемая карточка | 📋 планируется |
| **Phase 3 — Расширение** | map-фичи (кластеры/3D), table-фичи (колонки/фильтры), конструктор форм, вынос в отдельный репозиторий, desktop | 📋 бэклог |

```
Phase 0 ──✅──→ Phase 1 ──→ Phase 2 ──→ Phase 3
 Demo        Стабилизация  Реальные    Расширение
 (сейчас)                  данные
```

## Что блокирует переход к Phase 1

| 🔒 Блокер | Кто чинит | Подробнее |
|---|---|---|
| Виртуальный скролл таблицы (cold-empty quirk) | 🏗️ Команда фреймворка (`owner-web-ui`) | [Известные проблемы](known-issues.md#виртуальный-скролл-таблицы) |
| Анимация перехода между страницами (откат) | 🏗️ Нужна своя `Ui.Animate`-обёртка | [Известные проблемы](known-issues.md#анимация-перехода-между-страницами) |
| Typing-касты в виджетах и хэндлерах | 🛠️ App-команда + framework typing | [Технический долг](tech-debt.md#типизация) |

> [!NOTE]
> Часть блокеров — **в зоне фреймворка**, а не приложения. EWC их только потребляет: когда фреймворк починит виртуальный скролл и отдаст недостающие примитивы, в EWC меняется одна строка конфига. Эти пункты отмечены 🏗️.

## Как читать оценки в бэклоге

Оценки в [бэклоге](backlog/README.md) — **предварительные**, в человеко-днях, для одного разработчика, знакомого с Capsule. Это порядок величины (±50%), а не обязательство. Зависят от готовности бэкенда и фреймворковых примитивов.

| Шкала | Дней | Смысл |
|---|---|---|
| 🟢 S | 1–2 | Локальная фича в пределах приложения |
| 🟡 M | 3–5 | Несколько слоёв / новая механика |
| 🔴 L | 5+ | Кросс-пакетная работа / зависит от бэкенда |
