## 🧠 Что это за штука (за 30 секунд)

**Playground — мощный воркспейс для создания приложений на capsule.** Не демка и не Storybook — флагманский инструмент, который со временем съедает внешние сервисы (дизайн-тулзы, i18n-платформы, code-sandbox, тест-раннеры, билд/деплой).

Главная идея — **всё это одна система**:

> 💡 **Редакторы производят конфиги → платформа собирает из них продукт заказчика.**
> Дизайн, текст, UI, логика правятся визуально → выходят **per-tenant JSON** → билдер в контейнере собирает персональную сборку. Один язык, один поток.

```
редакторы (ui/style/text/logic)  →  per-tenant JSON-конфиги  →  forge (билд в контейнере)  →  продукт заказчика
        ▲                                    ▲                            ▲
   контракты компонентов              tenant-ось (заказчик)        WS-канал = управление + трейс
```

## 🎯 Позиционирование (зашито с первого дня)

- 🆓 **Бесплатный нейтральный юзерский инструмент.** Ноль рекламы, ноль «capsule»-брендинга в UI.
- 🎨 **Ребрендируемый.** Сам playground — это capsule-апп на web-style → лого/цвета/экосистема докидываются **branding-конфигом** позже. Никакого хардкода с самого начала.
- 📦 **Поставляется с фреймворком.** Юзер установил capsule → апп playground уже есть. Обновляется по **тому же каналу, что и все сборки** (priority 1).
- ⏳ Полный вариант «CLI пушит контейнер с кредами сервера» — позже. Сейчас простой.

## 👥 Роли и флоу работы

- 🧭 **Owner playground** — ведёт `apps/playground` + карту контрактов между пакетами, координирует.
- 🛠️ **Package-агенты** (под управлением пользователя) — пишут в своих пакет-скоупах.
- 🔁 **Цикл:** решаем что делаем → обсуждаем реализацию поверхностно → агент делает → **тестим вместе** → дальше. Внутренние моменты пакетов остаются на стороне исполнителя.
- 📏 Git-режим — `develop`-trunk + path-scoped commits (см. [`docs/_meta/parallel-dev-flow.md`](../_meta/parallel-dev-flow.md)).

## 🧩 Два новых фронт-пакета (всё остальное — subpaths или уже есть)

- **`@capsuletech/web-contract`** — leaf-протокол контрактов (zero-dep). Стюардит главный (это контракт/архитектура).
- **`@capsuletech/web-creator`** — один пакет, много subpath'ов (`/ui /style /text /logic /app /catalog /palette /tree /inspector /canvas /monitor /shell /data`).
- `web-renderer` — уже есть, остаётся отдельным.
- `forge` (Rust backend) — позже.

> 🧭 **Принцип:** не плодим пакеты под каждый редактор. Редакторы и общие тулзы — subpaths одного `web-creator`, настраиваются в аппе (паттерн «flow мини-апп + shell»). Отдельно держим только рендерер.
