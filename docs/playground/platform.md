## 🧠 Ментальная модель

> 💡 **Контейнер = capsule-воркспейс в коробке.** На вход — жирный исходник (база + каталог фич) + конфиг заказчика → билдер собирает персональную сборку (база + только купленные фичи). Плюс правка/тесты/запуск прямо в браузере.

> «В контейнере можно поднять и запустить что угодно.» Rust **оркестрирует**, тулзы — самодостаточны и модульны.

## 🔀 Текущее vs целевое

| | Сейчас (`docker/preview-server`) | Целевое (`forge`) |
|---|---|---|
| Что | статик-хост: локальный `capsule build` → tar → раздача | build + dev + test + правка в контейнере |
| Вход | dist-тарбол | **dist-тарбол ИЛИ репозиторий** |

**Два режима приёма:**
- 📦 **dist-тарбол** → только раздача (как сейчас).
- 📂 **репозиторий** → полный воркспейс: install → `capsule build` (+ entitlements) → раздача + dev + тесты + правка. Новый `/api/workspace`.

> ⚠️ install тяжёлый → жирный base-образ **пред-кеширует `@capsuletech/*`** (из Verdaccio). Сборка произвольного репо = произвольное исполнение → **ephemeral-изоляция** обязательна.

## 🧱 Композиция фич

- **Build-time (доставка заказчику)** — конфиг определяет, какие feature-модули **компилируются**. Тонкий артефакт, чужой код не утекает (IP/лицензии). Это `capsule build` + entitlements-конфиг.
- **В воркспейсе** держим **полный исходник** (для правки/тестов).

→ **Гибрид:** build-time на выход, полный исходник внутри.

**Доставка фич — два плеча:**
- 🧩 **baked-in** (build-time, дефолт).
- 🛰️ **federated** (web-remote runtime, включается по entitlement без ребилда).

## 🦀 forge — Rust-оркестратор (решения B1–B5)

| # | Решение |
|---|---|
| **B1** | новый крейт **специально под playground** (`backend/forge`) |
| **B2** | через Rust — **гибче**; рулит контейнерами (Docker API / bollard) |
| **B3** | **дирижёр**: тулзы самодостаточны/модульны (`capsule build`, `vitest`, `tsserver` — node-процессы), Rust только управляет |
| **B4** | **axum HTTP + WS** (WS для LSP / логов / тест-стрима / двустороннего канала) |
| **B5** | state-db (sqlite) для реестра воркспейсов; **лицензии пока без** |

**Изоляция:** **ephemeral per-tenant** контейнеры (лимиты CPU/RAM, таймауты, без доступа к хосту).

## 🏷️ Entitlements + tenant

- Конфиг заказчика — **per-tenant JSON**: набор фич + версии + **тема** (`theme.json`) + **текст** (`copy.json`) + брендинг.
- Это **тот же** customization-бандл, что производят редакторы.
- `tenant` = заказчик, сквозь intl/style/build. **v1 — single-tenant.**
- 🔏 **Подписанный конфиг** (лицензия от непроданных фич) — **позже**.

## 🎚️ Scope v1 и далее

| Этап | Что |
|---|---|
| **v1** | композиция-билд по конфигу + доставка (едет на priority 1 / preview-server) |
| далее | IDE (web-code container-backed) · тесты с виз · мониторинг воркспейса |
| позже | AI-правки (`backend/scriber` + `apps/agent`) · multi-tenant · лицензии · multi-target (desktop) |

> 📌 **Priority 1** (доставка билдов) идёт на **существующий preview-server** — forge не нужен. forge = **priority 4**.
