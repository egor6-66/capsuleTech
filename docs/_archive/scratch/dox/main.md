1. Философия: "The UI is a Shadow"HCA — это архитектурный фреймворк, где интерфейс (Entity) является лишь «глухой» проекцией логики. Вся власть сосредоточена в Контроллерах, которые управляют поведением через систему мета-тегов и реактивных цепочек.2. Стек и технологииRuntime: Solid.js (Fine-grained reactivity).Logic Engine: XState (Finite State Machines).Type System: TypeScript (Strict Contracts).Binding: Proxy-based event interception & Meta-tagging.3. Архитектурные слоиEntities (Stateless UI): Изолированные компоненты. Не содержат логики и не импортируют внешние зависимости. Общаются с миром только через data-meta атрибуты.Controllers (Behavior): Обертки, работающие на XState.Proxy-Interception: При монтировании Controller оборачивает Entity в Proxy, автоматически подписываясь на события (onClick, onInput) всех помеченных элементов.Tag Operations: Использование методов match(), pick(), omit() для управления группами элементов по их тегам и алиасам.Features (Domain Logic): Слой побочных эффектов. Только здесь разрешены запросы к API и интеграции. Feature общается с Контроллерами через асинхронный метод next().Widgets & Pages: Слой композиции, где конкретные Entity связываются с конкретной логикой.4. Механика взаимодействияDownstream (State): Состояние из XState-машины через Context и сигналы Solid точечно обновляет атрибуты Entity. Благодаря гранулярности Solid, перерисовываются только конкретные узлы.Upstream (Events): Событие из Entity перехватывается Proxy-слоем Контроллера. Контроллер видит target, meta-tags и текущий state, принимает решение и, если нужно, передает запрос выше через next().Tagging System: Система тегов и алиасов позволяет группировать элементы. Например, алиас @inputs может включать в себя теги email, password и phone.5. Золотой регламент (Compliance)No Upward Imports: Слой ниже никогда не импортирует слой выше.Entity Isolation: Entity не знает о существовании XState или API.Composition Only in Widgets: Запрещено жестко импортировать одну Entity в другую (например, Input в Form) — только через передачу в слоты на уровне виджета.


6. Строгая вертикальная изоляция (No Horizontal Imports)Внутри HCA действует правило абсолютной вертикали. Запрещены не только восходящие, но и горизонтальные импорты в пределах одного слоя:Изоляция сущностей: Entity.Button не может импортировать Entity.Icon. Все зависимости между ними разрешаются исключительно через Composition (слоты) на уровне Widget.Независимость логики: Controller.Auth не может знать о существовании Controller.Validation. Если им нужно взаимодействовать, они объединяются в Chain (цепочку), где коммуникация идет через посредничество вышестоящей Feature или родительского контекста.Цель: Любой компонент любого слоя должен быть «вырываем». Вы можете заменить одну реализацию другой или перенести её в другой проект, не потянув за собой шлейф из соседних файлов.
