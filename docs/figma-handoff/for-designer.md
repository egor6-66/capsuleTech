## 🧠 1. Ментальная модель за 30 секунд

Дизайн-система стоит на **токенах** — переменных дизайна (цвет, отступ, скругление, шрифт, тень). Из токенов собираются **компоненты** (кнопки, поля, карточки). Из компонентов — **экраны**.

Два слоя токенов:

- 📏 **Scale (общий слой)** — отступы, размеры шрифта, межстрочные, скругления, кривые анимаций, статусы. Один на всю систему. Файл: `tokens/scale.json`.
- 🎨 **Skin (тема)** — цвета, шрифты, тени. Меняется от темы к теме. Файлы: `tokens/themes/*.json`.

Темы (Zen, Black и т.д.) отличаются **только skin-слоем**. Смена темы меняет цвета, но раскладка/отступы/ритм те же. Это и есть «система».

### 🎯 Цвет = роль, а не оттенок

Главное отличие системного дизайна от «рисования». Цвета названы по **роли**, не по виду:

| 🎨 Роль | Где применять |
|---|---|
| `background` / `foreground` | основной фон страницы / основной текст на нём |
| `card` / `card-foreground` | фон карточки / текст в карточке |
| `primary` / `primary-foreground` | главное действие (кнопка) / текст на нём |
| `secondary` · `accent` | второстепенные действия / акценты |
| `muted` / `muted-foreground` | приглушённый фон / подписи, мелкий текст |
| 🔴 `destructive` | опасные действия (удалить), ошибки |
| `border` · `input` · `ring` | рамки / поля ввода / фокус-обводка |

Каждый «фон» имеет пару `-foreground` — цвет текста, который гарантированно читается на этом фоне. Всегда берётся пара.

## ⚙️ 2. Как завести токены в Figma (Tokens Studio)

1. 🔌 Установить плагин **Tokens Studio for Figma** (Figma → Plugins → найти «Tokens Studio»).
2. 📥 В плагине: **Import** → загрузить `scale.json`, затем `themes/zen-light.json` и `themes/zen-dark.json` (каждый станет отдельным «token set»).
3. 🌗 Собрать **Theme** «Zen» с двумя режимами:
   - ☀️ **Light** = `scale` + `zen-light`
   - 🌙 **Dark** = `scale` + `zen-dark`
4. 🪄 **Create variables / Export to Figma** — токены станут нативными Figma **Variables** с модами Light/Dark.
5. ✅ В макетах цвет выбирается не пипеткой, а **переменной** (`color/primary` и т.д.) — при переключении мода всё перекрашивается само.

> 💬 Для другого плагина или нативного импорта Variables — даётся версия под него.

## 🎨 3. Палитры и режимы — менять цвета целиком

Цвет в системе переключается не вручную по слоям, а **целыми наборами**. Две оси:

- 🌗 **Режим (mode)** — Light / Dark *внутри одной палитры*. Роли те же, меняются значения.
- 🎨 **Палитра (тема)** — целый набор цветов (Zen, Black, …). Отдельный skin-набор. Палитр может быть сколько угодно.

> 💡 Экран собирается из **ролей** (`primary`, `card`, `muted`…) **один раз**. Смена режима или палитры перекрашивает всё автоматически — перерисовывать ничего не нужно. В этом вся суть системы.

### ➕ Как добавить новую палитру

1. 🎯 Взять готовую палитру (например на **tweakcn** — покрутить и **Export**, отдаёт набор shadcn-токенов) или собрать свою.
2. 📄 Создать новый token set по образцу `zen-light` / `zen-dark` — **те же имена ролей**, другие значения.
3. 🌗 В Tokens Studio собрать новую **Theme**: Палитра × Light/Dark.
4. 🪄 **Push to Figma** → появляется новый mode. В макете переключение мода = смена палитры.

> ⚠️ Палитра меняет **только** значения ролей (skin: цвета/шрифты/тени). Общий слой (отступы, размеры, скругления) — единый для всех палитр и **не трогается**. Именно поэтому любую палитру можно «положить поверх» и всё останется консистентным.

> 🧰 Заметка по Figma: режимы (modes) на бесплатном плане ограничены — много палитр сразу могут потребовать платный тариф. Tokens Studio при этом хранит все палитры независимо от плана.

## 🔗 4. Ресурсы — где смотреть, чем вдохновляться

### 🧭 Понять, как сегодня делают веб-UI
- **shadcn/ui** — https://ui.shadcn.com — компонентный канон, на котором построена наша система. Как выглядят и ведут себя реальные компоненты.
- **tweakcn** — https://tweakcn.com — живой редактор тем shadcn (есть AI-таб). Цвета/радиусы/шрифты с мгновенным превью. Наши токены — ровно эта система.
- **Tailwind CSS** — https://tailwindcss.com — utility-система, на которой держится вся вёрстка (разделы spacing/colors/typography).
- **Refactoring UI** — https://www.refactoringui.com — золотой стандарт «как сделать, чтобы выглядело профессионально». Маст-рид.
- **Laws of UX** — https://lawsofux.com — принципы восприятия интерфейсов.

### 🌈 Цвет
- **OKLCH picker** — https://oklch.com — наше цветовое пространство.
- **Realtime Colors** — https://www.realtimecolors.com — примерка палитры на живом лейауте.
- **Radix Colors** — https://www.radix-ui.com/colors — образец доступных системных палитр.
- **UI Colors** — https://uicolors.app — генератор Tailwind-палитр.

### ✍️ Типографика
- **Type Scale** — https://typescale.com — подбор гармоничной шкалы размеров.
- **Google Fonts** — https://fonts.google.com — источник шрифтов (Inter, Playfair Display, JetBrains Mono и др.).

### 🧩 Компоненты и Figma-киты
- **Figma Community** — https://www.figma.com/community — поиск «shadcn ui kit», «shadcn design system». Готовые наборы под наш же канон.
- **Kobalte** — https://kobalte.dev — поведение компонентов (состояния, доступность). Компонент — это не картинка, а набор состояний.

### 👀 Насмотренность / тренды
- **Mobbin** — https://mobbin.com — паттерны реальных продуктов.
- **Godly** — https://godly.website — современная веб-эстетика.
- **Land-book** — https://land-book.com — лендинги.
- **Awwwards** — https://www.awwwards.com — тренды и награды.

### 🎛️ Про токены и Figma Variables
- **Tokens Studio** — https://tokens.studio · доки https://docs.tokens.studio
- **Figma Help** — https://help.figma.com — поиск «variables», «modes».

## 🤖 5. Промпты для ИИ — учиться и обсуждать

Промпты заточены под нашу систему и веб-дизайн в Figma — копируются и вставляются в чат с ИИ.

**🟦 Разобраться в основах:**
- «Объясни простыми словами, что такое design tokens и чем системный подход к цвету (роли: primary/secondary/muted/accent) лучше подбора оттенков вручную.»
- «Что такое пара "фон + foreground" в дизайн-системах shadcn и как ей пользоваться, чтобы текст всегда читался?»
- «Объясни modes (Light/Dark) в Figma Variables на примере и как организовать переменные для переключения темы одной кнопкой.»

**🟩 Прокачать раскладку и ритм:**
- «Что такое 8-point spacing grid? Покажи применение шкалы 4/8/12/16/24/32 px в Figma Auto Layout на примере карточки.»
- «Дай чек-лист хорошего Auto Layout: padding, gap, выравнивание, hug/fill — с объяснением каждого.»

**🟨 Собрать экран из системы:**
- «Есть дизайн-система с ролями цвета: background, card, primary, secondary, muted, accent, destructive (+ foreground-пары). Помоги собрать карточку товара / форму логина / дашборд, используя ТОЛЬКО эти роли. Обоснуй каждый выбор.»

**🟥 Получить критику:**
- «Вот скрин макета. Разбери по пунктам: визуальная иерархия, контраст и читаемость, консистентность отступов, выравнивание, типографика. Что улучшить?»
- «Проверь контраст текста и фона по WCAG. Где не хватает и как поправить, не выходя за роли цвета?»

**🟪 Понять связку с кодом:**
- «Объясни соответствие Tailwind utility-классов (bg-primary, p-4, rounded-lg, text-2xl) дизайн-токенам, чтобы называть слои так же, как в коде.»

## 📖 6. Словарь и теги — общий язык

Знание этих терминов = быстрый старт и понятный диалог с разработчиком и ИИ. Работают и как **поисковые теги**, и как темы для изучения.

| 🏷️ Термин | Что значит |
|---|---|
| **Design tokens** | Именованные переменные дизайна (цвет/отступ/радиус…), единый источник правды |
| **Design system** | Набор токенов + компонентов + правил, дающий консистентность |
| **shadcn / shadcn/ui** | Канон компонентов, на котором построена наша система |
| **Tailwind** | Utility-CSS-система; токены = классы (`bg-primary`, `p-4`) |
| **Semantic color / роль цвета** | Цвет назван по назначению (`primary`), а не по виду (`blue-500`) |
| **Foreground pair** | Парный цвет текста, читаемый на данном фоне |
| **OKLCH** | Современное цветовое пространство (наши цвета в нём) |
| **Token set / mode / theme** | Механики Tokens Studio и Figma Variables для тем и Light/Dark |
| **Auto Layout** | Авто-раскладка Figma (flex-аналог); padding/gap/hug/fill |
| **8pt grid / spacing scale** | Шкала отступов кратно 4/8 для ритма |
| **Type scale** | Гармоничная шкала размеров шрифта |
| **Elevation / shadow** | Глубина через тени (`shadow-sm…2xl`) |
| **Radius scale** | Шкала скруглений (`radius-sm…3xl`) |
| **Component variants** | Состояния/виды компонента (size, intent, state) |
| **Density / compact** | Сжатие отступов под компактный режим (runtime) |
| **Contrast / WCAG** | Норматив читаемости текста на фоне |
| **Atomic design** | Подход «атомы → молекулы → организмы» к UI |

**🔖 Готовый набор тегов для поиска и ИИ:**
`design tokens` · `design system` · `shadcn ui` · `tailwind css` · `figma variables` · `figma auto layout` · `light dark mode` · `semantic colors` · `8pt grid` · `type scale` · `OKLCH` · `accessibility WCAG` · `component variants` · `web ui patterns`

---

> 🚀 Насмотренность через ресурсы, практика через промпты, сборка в Figma из переменных системы. Если в системе чего-то не хватает или что-то неудобно — это повод доработать токены в коде, а не подгонять код под обходные пути. Так дизайн ведёт, а система подстраивается.
