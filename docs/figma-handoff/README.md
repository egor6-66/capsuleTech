# 🎨 Design System — Figma Handoff

> **Навигация:** 📍 [Общее](README.md) · 🎨 [Дизайнеру](for-designer.md) · 🛠️ [Разработчику](for-developer.md) · 📐 [Контракт](CONTRACT.md) · 🎛️ [Токены](tokens/)

Стартовый пакет для переноса дизайн-системы в Figma: токены (цвета, отступы, скругления, шрифты, тени, анимации) и контракт о том, как держать дизайн и код **в одном языке**.

Главная идея:

> 💡 **Источник правды — код.** Дизайн собирается поверх реальных токенов и компонентов, а не подбирается на глаз. Меняется токен в коде → обновление приезжает в Figma. Экран собран из переменных системы → разработчик забирает его почти 1:1.

## 📦 Состав пакета

```
figma-handoff/
├── 📍 README.md          ← общее, точка входа
├── 🎨 for-designer.md    ← гайд дизайнера + ресурсы + промпты
├── 🛠️ for-developer.md   ← связь токенов с кодом
├── 📐 CONTRACT.md        ← правила именования + «skin-контракт»
└── 🎛️ tokens/
    ├── scale.json        ← 📏 общий слой (отступы, скругления, шрифты, анимации, статусы)
    └── themes/
        ├── zen-light.json   ← ☀️ тема «Zen», светлый режим
        └── zen-dark.json    ← 🌙 тема «Zen», тёмный режим
```

- 📏 `scale.json` — общий для всех тем (не меняется).
- 🎨 `themes/` — «шкурки»: только то, что отличает одну тему от другой.

> ℹ️ Сейчас в пакете **одна тема (Zen) как эталон** — на ней отлаживается флоу импорта. После согласования формата остальные темы догенерируются в той же структуре.

## 🧩 Формат токенов

Файлы — в формате **DTCG** (W3C Design Tokens) — общий язык токенов, который понимает плагин **Tokens Studio** для Figma. Для другого плагина или нативных Figma Variables — версия даётся под него.

## 🌈 Про цвета

В коде цвета — в **OKLCH** (современное цветовое пространство). Figma его пока не умеет нативно, поэтому здесь цвета уже переведены в **hex**. Возможны крошечные расхождения оттенка с браузером — это нормально и ожидаемо.
