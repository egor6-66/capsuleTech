«Договор» между Figma и кодом. Короткий, но именно он даёт **синхрон**: дизайн и реализация говорят одними и теми же словами.

---

## 🔒 Правило №1 — имена не переименовываются

Имена токенов (`primary`, `card`, `muted-foreground`, `radius-lg`, `space-4`…) — это **контракт с кодом**. В коде те же имена используются как Tailwind-классы (`bg-primary`, `rounded-lg`, `p-4`).

- ✅ Можно: менять **значение** токена (другой оттенок `primary`, другой `radius`).
- ❌ Нельзя: переименовывать токен, выдумывать имена вне набора, плодить синонимы.

> ⚠️ Если `primary` переименовать в `brand` — автоматический перенос макета в код ломается, связь рвётся.

## 🧱 Правило №2 — две зоны токенов

| Зона | Что это | Меняется между темами? |
|---|---|---|
| 🎨 **Skin (тема)** | Цвета, скругление-якорь, шрифты, тени | ✅ Да — это и есть «тема» |
| 📏 **Scale (общий слой)** | Отступы, шкала шрифта, межстрочные, трекинг, кривые анимаций, статусы | ⚠️ Обычно нет — общее для всех тем |

Темы отличаются друг от друга **только skin-слоем**. Шкала отступов/типографики — одна на всю систему (так интерфейсы остаются консистентными).

## 🎨 Правило №3 — набор цветов фиксированный (канон shadcn)

Цветовой набор — **канонический набор shadcn/Tailwind**. Он специально стандартный, чтобы:

- 🌍 любой стиль/палитру из веб-комьюнити (например с tweakcn) можно было «положить поверх» и всё заработало;
- 🧩 готовые Figma-киты на shadcn ложились без переделок.

Полный список цветовых токенов (каждый — пара «фон» + `-foreground` для текста на нём):

```
background / foreground
card / card-foreground
popover / popover-foreground
primary / primary-foreground
secondary / secondary-foreground
muted / muted-foreground
accent / accent-foreground
destructive / destructive-foreground
border · input · ring
chart-1 … chart-5
sidebar (+ foreground / primary / accent / border / ring)
```

Плюс семантические статусы (в общем слое): 🟢 `success`, 🟡 `warning` (+ их `-foreground`). Для ошибки — 🔴 `destructive`.

> 🚫 Новые цветовые токены не добавляются вручную. Нужен новый цвет — заявка разработчику, заводится в коде и приезжает в Figma. Так набор остаётся каноном.

## 🚫 Правило №4 — что НЕ переносится в Figma

Часть вещей живёт только в коде:

- ⏱️ **Длительности анимаций** (200ms, 320ms…) — это поведение, не визуальный токен. В Figma переносятся только **кривые** (easing).
- 📐 **Density / calc** — отступы умеют сжиматься (компактный режим) на лету. В Figma приходит «снимок» при density = 1.
- 🧪 **Градиенты, скроллбары, переходы** — внутренние, не часть дизайн-контракта.
