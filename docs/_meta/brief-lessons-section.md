# Brief: Раздел Lessons (архитектура)

> [!abstract] Проектируем раздел аппа **Lessons**. Гипотеза юзера: делим на (1) концепты/философия — прозой, (2) правила/грамматика — структурой. Ниже: оценка гипотезы + уточнение (третья нога — практика) + модель. Связано с [[brief-app-architecture]], [[brief-learning-concepts]], [[word-as-image]].

## Гипотеза юзера
- **Часть A** — концепты, идеи, подходы, философия. Форма — **проза**, «описать, как думать».
- **Часть B** — правила: грамматика, времена, глаголы. Форма — **структура**, как текущий `grammar/`.
- Вопрос: годится ли такое деление.

## Оценка: деление верное — это ось «мышление vs справочник»

| | **Концепты** (mindset) | **Правила** (reference) |
|---|---|---|
| форма | проза, нарратив | таблицы, lookup |
| количество | мало (~10–15), ядро | много, растёт всегда |
| как юзают | прочитал → сдвинул мышление | сверяешься при сборке фразы |
| жизненный цикл | редко меняются | пополняются постоянно |
| что дают | КАК думать | КАК правильно |
| примеры | слово=образ, две оси, `be` не опускается, чанки, связная речь | времена, do-support, местоимения, артикли |

Две разные сущности по форме, объёму и использованию → да, разделять.

## Чего не хватает: третья нога — Практика (Drills)
- Концепты + правила = **пассивное** знание. Навык даёт **делание**.
- **Drill** = RU→EN production + точечный фидбек + **возврат к граблям** (весь наш живой формат).
- Канон: **урок заканчивается деланием, а не чтением.**

## Ключевая связка: Lesson = путь сквозь библиотеки
- **Концепты** и **Правила** — это ДВЕ БИБЛИОТЕКИ переиспользуемого контента.
- **Lesson** — юнит, который **сшивает**: концепт(ы) → правило(а) → дрил(ы).
- **DRY:** одно правило («времена») тянут десятки уроков → храним раз, ссылаемся. То же, что Song→refs и «слово = join-key» из [[brief-app-architecture]], только **Lesson→refs**.

```
Концепты (lib) ─┐
Правила   (lib) ─┼─►  Lesson (путь) ──► Drill (делание)
Дрилы     (lib) ─┘
```

## Структура (vault → апп)
| Апп | Vault | Статус |
|---|---|---|
| Концепты (философия) | `lessons/concepts/*.md` (проза) | seed: [[word-as-image]]; философия из [[brief-learning-concepts]] |
| Правила (reference) | `grammar/*`, `phonetics/*`, `speech/*` | **уже есть**, переиспользуем |
| Дрилы | `methods/exercises.md` → пер-тема | есть заготовка |
| Урок (путь) | `lessons/*.md` — ссылки на концепты+правила+дрилы | новое |

> [!note] `brief-learning-concepts.md` — это **дизайн-док для нас** (педагогика), а learner-facing концепты — **отдельные заметки** (как [[word-as-image]]). Не путать: бриф проектирует, концепт-заметка учит.

## Модель данных (черновик)
```
Concept { id, title, prose, principle, examples[], relatedRuleIds[] }
Rule    { id, topic, body(tables|prose), examples[] }      // ≈ текущий grammar/
Drill   { id, promptRu, answerEn, tests[tag], graboTag? }  // graboTag = типовая грабля
Lesson  { id, title, level, conceptIds[], ruleIds[], drillIds[] }

Lesson ──refs──► Concept | Rule | Drill   (переиспользование, не копия)
```

## Сквозные линии (через всё)
Образы · песни как якорь · точечный фидбек · **spaced-возврат к личным граблям** (`graboTag`) · инлайн-фонетика.

## Открытые вопросы (на подтверждение)
1. **Библиотеки + пути (гибрид)** vs две плоские категории? Рекомендую гибрид: Концепты/Правила как переиспользуемые либы, Lesson — путь сквозь них.
2. **Переиспользуем `grammar/` как «Правила»** vs зеркалим в `lessons/rules/`? Рекомендую переиспользовать — `grammar/` уже и есть «Правила».
3. Нужен ли уровень (L0–L5) на **Lesson**, или только на слове (как решили в [[brief-app-architecture]])? Вероятно на Lesson тоже — для порядка прохождения.
