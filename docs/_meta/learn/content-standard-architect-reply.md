# Ответ архитектора по стандартам контента

> [!info] core + concept стандарты **приняты** (зелёный). Модель наследования (core-тело →
> профили) — то, что надо. Ниже: решения по 4 рекомендациям + два факта движка, которые уточняют
> MEDIA и callout'ы. Что помечено ✅ — вписывай в [[core-content-standard]] + чек-лист валидатора.

## Факты движка (важно для стандарта)
1. **Реестр callout'ов.** Наш рендер (`marked`-extension) рисует **`info · tip · warning · note`**.
   `todo` пока **нет** — он молча падает в `note`. Решение: **я добавляю `todo` в движок**
   (KNOWN_CALLOUTS + стиль в Prose). Итоговый закрытый реестр = **`info · tip · note · warning · todo`**
   ✅ — как ты и заложил. Новый тип и дальше — только через меня (архитектор заводит стиль).
2. **Callout без иконки** — различие только цветом рамки/фона/заголовка (контент = innerHTML, туда
   Solid-иконку не вставить). Это ок; просто знай, что глиф-иконки в плашке не будет.
3. **MEDIA — авто-по-id работает НЕ везде.** 🔊/образ цепляются по id **только в структурных
   блоках** (тайл слова, чип дрилла — читают `sense.audio.url`/`image`). В **markdown-теле** строка
   `go /ɡoʊ/ — идти` — просто проза, механики подстановки 🔊 к слову там **нет**.

## Решения по рекомендациям
| # | вопрос | решение архитектора |
|---|---|---|
| 1 | MEDIA: явный маркер или авто-по-id | ✅ **авто-по-id, автор ничего не рисует.** Но: авто-медиа = только структурные словные единицы. **Инлайн-🔊 в markdown-теле — вне MVP** (будущий word-ref: `[[w:go]]` → чип по id, зеркало wikilinks; заведу когда дойдём). Явный MEDIA-блок для не-словных якорей (аудио фразы) — **тоже отложить**, пока нет реального кейса. В теле автор медиа НЕ помечает. |
| 2 | провенанс обязателен | ✅ **да, для модерируемого контента.** `author` + `updated` — **обяз.**; `reviewed_by` — опц. (пусто = не отревьюено). Вписывай в [[authoring-contract-for-teacher]]. Импортёр примет поля — заведу мини-фикс (backend-lang schema), помечу. |
| 3 | `updated` или `version` | ✅ **`updated` (ISO-дата) на MVP.** `version: N` — отложен до миграции прогресса (тогда вернёмся). |
| 4 | эмодзи-смысл через компоненты | ✅ **да.** В теле — только **декоративные** эмодзи. Смысловые (🔊 «озвучить», ✅ «верно», ⚠) — **через компоненты кита** (у иконки кнопки/вердикта есть текст-лейбл, a11y). Это ровно наш component-model канон. Валидатор: значимый глиф в сыром теле = предупреждение. |
