# Стандарт контента — ЯДРО (структура тела)

> [!info] Дополняет [[authoring-contract-for-teacher]] (SSOT по frontmatter и идентичности файла). Контракт задаёт ШАПКУ; этот стандарт — СТРУКТУРУ ТЕЛА (то, что контракт оставляет как «проза markdown»). Наследуется всеми профилями (rule/concept/lesson/topic/doc). Валидатор = core-проверки тела + проверки профиля.

## Скелет тела (общий для всех типов)
`H1 (= title)` → ровно один интро-`[!info]` (1–2 строки) → блоки → (опц.) `[!todo]`.
- Нет «голого» текста вне блоков.
- H2/H3 — только заголовки блоков.
- В заголовках **нет нумерации** (порядок несёт `order` из frontmatter).

## Грамматика блоков (домен-агностик, закрытый список)
Профиль выбирает подмножество; может специализировать (напр. `rule.SOUND`).

| блок | когда | форма |
|---|---|---|
| **PRINCIPLE** | суть правила/идеи | одна **жирная строка** под заголовком |
| **EXAMPLES** | плоские иллюстрации | список; один пункт = одна единица; канон-строка |
| **CONTRAST** | X vs Y / мин. пары | таблица ровно 2 колонки |
| **MATRIX** | парадигма / сетка | таблица с шапкой строк и колонок |
| **STEPS** | процедура «как применять» | нумерованный список |
| **NOTE** | оговорка/исключение | callout `note`/`warning` |
| **MEDIA** | якорь озвучки/образа | плейсхолдер 🔊/картинка (движок подставит) |

## Форматы строк (машинно-проверяемые)
- Строка-пример: `<term> [/IPA/] [— перевод]`.
- Несколько единиц в одной строке — разделитель ` · ` (middot), **НЕ запятая**.
- Тире `—` перед переводом; IPA в слэшах `/…/`; термин-код в `` `…` ``.

## Анти-бардак (общие правила)
1. **Один тип контента → один блок → один формат.** В пределах темы всё однотипное выглядит одинаково.
2. **Таблица** (параллельные данные, у всех те же колонки) **vs список** (плоские иллюстрации). **НИКОГДА** >1 параллельной единицы в одном буллете.
3. Нет текста вне блоков; ровно один H1; ровно один интро-`[!info]`.
4. **Маршрутизация:** контент живёт в одном каноне; остальные **ссылаются** (`[[id]]`), не дублируют.

## Wikilinks — гигиена
- В телах импортируемого контента — **ТОЛЬКО на импортируемые id** (rule / concept / drill / lesson). НЕ на `briefs/`, `methods/`, `journal/` (в аппе мертвы) → [[wikilinks-hygiene-for-teacher]].
- Валидатор: `[[…]]` на неимпортируемый/несуществующий id в теле = предупреждение.

## Callout-реестр (закрытый — то, что рисует движок)
`info` (интро/контекст) · `tip` (совет) · `note` (оговорка) · `warning` (важно / ошибкоопасно) · `todo` (что дополнить).
Новый тип — только через реестр (архитектор заводит стиль), самодеятельность не проходит.

## Core-чек-лист валидатора (тело)
1. Ровно один H1 (= title) и один интро-`[!info]`; нет текста вне блоков.
2. Каждая секция — блок из грамматики выше; таблица/список по правилу анти-бардака.
3. Строки-примеры — канон-формат (middot-разделитель, тире перед переводом, IPA в слэшах).
4. Заголовки без нумерации.
5. Wikilinks тела — только на импортируемые id.
6. Callout'ы — только из реестра.

*(Каждый профиль добавляет свои проверки поверх этого.)*
