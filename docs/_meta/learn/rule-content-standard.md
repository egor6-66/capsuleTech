# Эталон оформления rule-контента (грамматика · фонетика)

> Стандарт для авторов/модераторов и для скриптов валидации. Цель: **один тип контента → один тип блока → один формат**. В рамках темы все однотипные пункты выглядят одинаково. Всё, что не подходит под блок из списка ниже, — не проходит валидацию.

## 0. Область
Применяется к докам с `type: rule` (сейчас `category: grammar` и `category: phonetics`). Датасет слов (Word/Sense) — отдельный стандарт (там своё поле `pron_ru`); здесь речь только про справочные rule-доки.

## 1. Скелет документа (жёсткий порядок)
1. **Frontmatter** (см. §2).
2. **H1** — заголовок темы (ровно один, без номера в тексте — порядок задаёт `order`).
3. **Интро-callout** `> [!info]` — 1–2 строки: о чём тема + ссылки. Ровно один.
4. **Блоки** — последовательность секций, каждая = один тип блока из §4.
5. (опц.) финальный `> [!todo]` — что дополнить.

Ничего между H1 и интро-callout. Никакого «голого» текста вне блоков.

## 2. Frontmatter (схема валидации)
```yaml
tags: [english, <category>]     # обяз., список
type: rule                      # обяз., ровно "rule"
category: grammar | phonetics   # обяз., из закрытого списка
order: <int>                    # обяз., порядок в разделе (10,20,30…)
status: skeleton | in-progress | done   # обяз.
lessonTag: <kebab-slug>         # обяз., = имя файла без расширения
priority: high | normal | low   # опц.
prerequisites: [<lessonTag>…]   # опц.
```
Валидатор: все обязательные поля на месте; `category`/`status`/`priority` — из закрытых списков; `lessonTag` == имя файла; `order` уникален в пределах `category`.

## 3. Произношение — политика (US General American, IPA-only)
- Транскрипция **только IPA**, в слэшах `/…/` (фонемно). **Без кириллицы** в rule-доках.
- Диалект — **General American (rhotic)**. Закрытый набор символов ниже; британских (`ɜː`, `ɒ`, `ə` вместо r-окраски, немой конечный `r`) быть не должно.

**Разрешённые символы (GenAm):**
```
Согласные:  p b t d k ɡ  tʃ dʒ  f v θ ð s z ʃ ʒ h  m n ŋ  l r j w
Гласные:    iː ɪ ɛ æ ɑ ɔ ʊ uː ʌ ə   ɝ (уд. r-гласный)  ɚ (безуд. r-гласный)
Дифтонги:   eɪ aɪ ɔɪ oʊ aʊ
R-окраска:  ɑr ɔr ɛr ɪr ʊr
Ударение:   ˈ (главное)  ˌ (второстепенное)
```
Примеры канона: work `/wɝk/`, letter `/ˈlɛtɚ/`, think `/θɪŋk/`, go `/ɡoʊ/`, car `/kɑr/`.
Валидатор: каждый `/…/` состоит только из символов набора; ловит `ɜː`, `ɒ`, конечный немой `r` без гласного, латиницу внутри слэшей.

## 4. Типы блоков (закрытый список)
Любая секция — ровно один из пяти. Шаблоны строгие.

### 4.1 RULE — правило + иллюстрации
Когда: одно правило и плоский список примеров на него.
```markdown
## <Заголовок правила>
**<Принцип одной строкой.>**
- <пример-строка>
- <пример-строка>
```
Валидатор: `##`-заголовок; ровно одна жирная строка-принцип сразу под ним; далее только буллеты-примеры канон-формата (§5).

### 4.2 CONTRAST — оппозиция / минимальные пары
Когда: «X vs Y», две колонки, попарно.
```markdown
## <X> vs <Y>
| <X> | <Y> |
|---|---|
| <term /IPA/> | <term /IPA/> |
```
Валидатор: заголовок содержит `vs`; таблица ровно 2 колонки; в ячейках — термы с `/IPA/`.

### 4.3 MATRIX — парадигма / сетка
Когда: таблица-сетка с осмысленной шапкой строк и колонок (время×вид; правила -s/-ed; долгие/краткие).
```markdown
## <Заголовок>
| <шапка> | <колонка> | <колонка> |
|---|---|---|
| <строка> | … | … |
```
Валидатор: ≥2 колонки данных + шапка; строка-разделитель на месте; пустая строка перед таблицей.

### 4.4 SOUND — один звук (только phonetics)
Когда: один фонемный блок. **Один блок = ровно один звук.**
```markdown
### /<IPA>/ — <имя/тип>
<Артикуляция одной строкой.>
- Пары: <term /IPA/> — <term /IPA/> · <term /IPA/> — <term /IPA/>
- Примеры: <term /IPA/> — <перевод> · <term /IPA/> — <перевод>
```
Валидатор: `###`-заголовок начинается с `/IPA/`; одна строка артикуляции; строки `Пары:`/`Примеры:` — опц., но если есть, то канон-формат. Запрещено перечислять >1 звука в одном блоке.

### 4.5 NOTE — оговорка
Когда: caveat/исключение/ссылка-предупреждение.
```markdown
> [!note] <текст>
> [!warning] <текст>
```
Валидатор: только callout-типы `note|warning|info|todo`.

## 5. Канонический формат строки-примера (машинно-проверяемый)
Единый вид примера с произношением и/или переводом:
```
<term> /<IPA>/ — <перевод/пояснение>
```
- `/IPA/` — обязателен, если строка про произношение.
- `— <перевод>` — обязателен, если строка про значение.
- Разделитель нескольких примеров в одной строке — ` · ` (middot), не запятая.

Регулярка (ориентир): `^\S.*\s/[^/]+/(\s—\s.+)?$`

## 6. Правило «таблица vs список» (анти-бардак)
- **Параллельные данные** (у каждого пункта те же 2–3 атрибута) → **таблица** (CONTRAST/MATRIX).
- **Плоские иллюстрации** одного правила → **список**, **один пункт = один пример**.
- **Запрещено** несколько параллельных единиц в одном буллете (типичный баг: три дифтонга в одной строке → должно быть три SOUND-блока).

## 7. Маршрутизация контента (без дублей)
Контент живёт в **одном** каноническом доке; остальные **ссылаются**, не копируют.
- Отдельные звуки → `phonetics-sounds`.
- Связная речь (редукции, флэп, вставные) → `connected-speech`.
- Звук окончаний `-s`/`-ed` → MATRIX в `grammar-nouns` / `grammar-verbs-tenses` (в фонемном доке — только ссылка).
Валидатор (мягкий): предупреждает, если одна и та же MATRIX-таблица встречается в двух доках.

## 8. Чек-лист валидатора (сводка)
1. Frontmatter по схеме §2.
2. Ровно один H1; ровно один интро-`[!info]`; нет текста вне блоков.
3. Каждая секция — блок из §4, шаблон соблюдён.
4. Все `/…/` — только GenAm-символы §3 (нет `ɜː`, `ɒ`, латиницы, немого `r`).
5. Строки-примеры — канон §5 (middot-разделитель, тире перед переводом).
6. Нет >1 звука/единицы в одном буллете (§6).
7. `order` уникален в `category`; `lessonTag` == имя файла.

## Эталонный образец (фрагмент phonetics)

```markdown
---
tags: [english, phonetics]
type: rule
category: phonetics
order: 10
status: in-progress
lessonTag: phonetics-sounds
---

# Фонетика: отдельные звуки

> [!info] Один блок = один звук: артикуляция + минимальные пары. Связная речь → [[connected-speech]]; акценты → [[accents-us-uk]].

### /θ/ — глухое th
Язык между зубов, без голоса.
- Пары: think /θɪŋk/ — sink /sɪŋk/ · three /θriː/ — tree /triː/
- Примеры: think /θɪŋk/ — думать · both /boʊθ/ — оба

### /ð/ — звонкое th
Язык между зубов, с голосом.
- Пары: they /ðeɪ/ — day /deɪ/ · than /ðæn/ — Dan /dæn/
- Примеры: the /ðə/ — определённый арт. · that /ðæt/ — тот

### /oʊ/ — дифтонг
Скольжение «о→у», губы округляются.
- Примеры: own /oʊn/ — свой · go /ɡoʊ/ — идти · shone /ʃoʊn/ — светил

> [!note] Рус. ухо слышит /θ/ как с/ф/т — но это ОДИН звук: язык всегда между зубов.
```
