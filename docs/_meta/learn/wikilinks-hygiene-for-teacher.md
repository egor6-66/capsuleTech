# Wikilinks в учебных текстах: только на импортируемое

Живой факт из аппа: в теле `word-as-image` ссылка `[[brief-learning-concepts]]`
— клик уходит в никуда (в консоли: «ref не найден»). Это НЕ баг: `briefs/`,
`methods/`, `journal/` в апп не импортируются (финалочка) — ссылка на них в
learner-тексте мертва по построению.

## Правило (дополняет Амендмент №2 п.3)

В **телах** импортируемого контента (concepts / rules / drills / lessons)
wikilinks — ТОЛЬКО на id импортируемого: правила, концепты, дриллы, уроки.
Ссылки на брифы/методички — это «бриф проектирует, заметка учит» наизнанку:
ученику некуда по ним идти. В своих дизайн-доках ссылайся как угодно.

## Что сделать

Пройтись по телам импортируемых файлов (их пока немного: концепт + 14 правил +
9 дриллов + урок) и вычистить/заменить ссылки на неимпортируемое: либо убрать,
либо перенаправить на существующий id (напр. `[[brief-learning-concepts]]` →
по смыслу на конкретное правило/концепт, когда он появится).

На нашей стороне в бэклоге — «мёртвые» wikilinks рендерить приглушённым
текстом без клика (чтобы протечка не выглядела кликабельной), но гигиена
ссылок всё равно на авторе.
