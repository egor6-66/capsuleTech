# Фидбэк архитектора: все 4 решения приняты (Егор подтвердил)

> [!success] По [[lessons-model-teacher-notes]] — согласие по всем пунктам.
> Ниже: что решено, одна техническая фиксация по `pattern`, и что нужно от
> вас двоих для финалочки. Общаемся здесь же, в `docs/_meta/learn/`.

## Решено

1. **Скелет Drill принят как основа:**
   `promptRu + context? + answerEn + accept[] + nearMiss[] + graboTag`.
   `nearMiss` — самая ценная часть: именно она позволит аппу давать точечный
   фидбек («точку даёт Past Simple») по правилам, без нейронки. Канон авторства
   (однозначность через маркер ИЛИ context) принимаем как валидацию importer'а:
   дрилл без маркера и без context — **не импортируется, с понятной ошибкой**.

2. **Образ-мост — да, и он уже становится конвейером.** Поля `image` (образ на
   смысле) и мост на связи заводим в словарной базе. Параллельно (прямо сейчас)
   строится сервис генерации картинок: твой текст-образ → картинка автоматически,
   рядом со словом, как озвучка. Т.е. `word-as-image` = данные + автоматический
   визуал. Пока поля нет — картинки временно генерятся из самого слова, с
   образом станет точнее.

3. **Папка дриллов — `drills/` отдельно** (top-level). Дрилы — самостоятельная
   библиотека, третья нога. `methods/` остаётся вашими дизайн-доками — importer
   туда не смотрит.

4. **Уровень L0–L5 на Lesson — рукой**, отдельно от уровней слов. Подтверждено.

**Плюс решение по хранению:** весь lessons-контент живёт в той же базе, что и
словарь — дрилы прошиты словами насквозь, так они соединяются напрямую.
На тебя это влияет одним приятным следствием: **слово в дрилле = ссылка на
словарь** (если слова нет — сначала добавляем его в словарь, потом дрилл).

## Одна фиксация формата (важно для nearMiss)

`nearMiss[].pattern` — без хитрых языков описания. Два типа, указывается явно:

```yaml
nearMiss:
  - match: contains        # простое вхождение
    pattern: "did eat"
    hint: "Past Simple здесь ломает 'раньше чем' — нужен had + V3."
  - match: regex           # когда без него никак
    pattern: "\bhad\b.*\bcall\b"
    hint: "had относится к ДРУГОЙ половине: точку во времени даёт Past Simple."
```

Если `contains` покрывает 90% случаев из твоей практики — отлично, живём на нём.

## Что нужно для финалочки (после неё стройка начинается)

1. **Финальный набор полей** Concept / Rule / Drill / Lesson — свести в один
   документ здесь же (`lessons-model-final.md`). За основу — твой скелет Drill +
   модель из [[brief-lessons-section]].
2. **Первый эталонный дрилл-файл** `past-perfect-which-clause.md` — целиком,
   с frontmatter, как он будет лежать в vault. Он станет каноном формата,
   по которому пишется importer (и валидатор однозначности).
3. **Пути vault** — точные папки, куда смотрит importer: `lessons/`,
   `lessons/concepts/`, `drills/`, `grammar/` (и где физически лежит vault).
4. **Имена = id навсегда** — конвенцию из твоей докрутки 3 (kebab, тема-первой)
   вписать в финалочку как правило №0.
