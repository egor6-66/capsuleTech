# Как мы хотим хранить и кормить лексику — набросок для обсуждения

Привет! Мы собрали первый каркас лексической библиотеки и хотим начать наполнять её словами. Ниже — **как устроена база** и **как удобнее всего отдавать данные**. Это **набросок, не жёсткие рамки**: формат подстроим под тебя, главное — чтобы поля легли. Накидывай свои предложения (что важно для обучения, чего тут не хватает) — потом обсудим.

## Главная идея модели (по-человечески)

**1. Единица хранения — значение (sense), а не слово-строка.**
У одного слова бывает несколько значений с разной метой. Пример:
- `bank` → значение-1 «берег реки», значение-2 «финансовый банк».
У них разные синонимы, домены, уровень — поэтому каждое значение живёт отдельно.

**2. Теги — это группы, и они универсальны.**
Тег = смысловая/лексическая/доменная/фонетическая группа. Через теги мы и фильтруем, и подтягиваем «близкие слова». Виды тегов:
- `semantic` — смысловые группы (синонимы = общий semantic-тег, напр. `synset:glad` для happy/glad/joyful);
- `lexical` — грамматические;
- `domain` — тематика (`tech`, `finance`, `nature`…);
- `context` — ситуация употребления (`emotion`, `business`…);
- `phonetic` — звуковые фичи (`θ`, `ð`…), чтобы делать упражнения «все слова со звуком θ».

**3. Фасеты — по одному значению на каждое (для фильтра).**
Часть речи, уровень, регистр, частотность. Это атрибуты, у значения ровно один.

**4. Связи между значениями — типизированные.**
Синонимы делаем через общий semantic-тег (группа). А направленные связи — отдельно: `antonym` (антоним), `hypernym/hyponym` (шире/уже), `part_of` (часть↔целое).

## Какие поля нам нужны (чтобы данные легли)

На **каждое значение**:

| Поле | Обяз.? | Что это |
|---|---|---|
| **слово (написание)** | да | сама строка, напр. `bank` |
| **язык** | да | сейчас `en_US` (позже добавим другие) |
| **значение (глосса)** | желательно | короткое определение, чтобы различать значения одного слова |
| **часть речи** | да | noun / verb / adjective / adverb / … |
| **уровень** | опц. | A1…C2 (или твоя шкала — скажи какую удобнее) |
| **регистр** | опц. | formal / informal / neutral |
| **частотность** | опц. | насколько частое (можно «высокая/средняя/низкая» — переведём) |
| **теги** | желательно | список: имя + вид (semantic/domain/context/phonetic/lexical) |
| **связи** | опц. | список: тип (antonym/hypernym/…) + на какое слово/значение |
| **фонетика (IPA)** | опц. | если есть; иначе позже сгенерим автоматически |
| **примеры / заметки** | опц. | если хочешь приложить — скажи, заведём поля |

## Как это выглядит на примере

```
слово:       happy
язык:        en_US
значение:    feeling or showing pleasure
часть речи:  adjective
уровень:     A1
регистр:     neutral
теги:        synset:glad (semantic), emotion (context)
связи:       antonym → sad

слово:       bank
язык:        en_US
значение:    land along the side of a river
часть речи:  noun
уровень:     B1
теги:        nature (domain), geography (semantic)

слово:       bank
язык:        en_US
значение:    institution that keeps money
часть речи:  noun
уровень:     A2
теги:        finance (domain), institution (semantic)
```

(Это просто иллюстрация полей — отдавать можешь в любом удобном виде.)

## Как мы это «съедим»

- Ты отдаёшь в **удобном тебе формате** — на выбор: таблица (Excel/CSV), YAML-блоки как выше, markdown, Google-таблица. Что эргономичнее для наполнения.
- Мы пишем **переходник** под твой формат → в нашу базу. Под разные источники сделаем разные переходники, но **ты — основной**.
- **Повторная заливка безопасна**: обновляем существующее, дубли не плодим. Можно дозаливать порциями.
- **Проверяем на ошибки** при заливке (опечатка в части речи, тег которого нет и т.п.) → вернём тебе список, что поправить. Ничего молча не сломается.
