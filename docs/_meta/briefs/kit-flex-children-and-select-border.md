## Правка 1 — Flex-пресет должен показывать layout (с детьми)

**Что не так:** прошлый заход дал пустому Flex дашед-бордер + фон. Это бесполезно: Flex — layout-контейнер, его суть (`direction` / `wrap` / `gap` / `align` / `justify`) видна **только по детям**. На пустом контейнере кручение настроек инспектора не даёт визуального эффекта — нечего перестраивать.

**Что надо:** каждый Flex-пресет (`flex.presets.ts`) наполнить **2–3 дочерними плитками-плейсхолдерами**, чтобы при изменении настроек реально было видно перестроение.

- Дети — видимые нейтральные блоки. Рекомендация: `ui.Skeleton` (готовый placeholder-блок) или маленькие `ui.Card` с номером, либо `ui.Typography` «1/2/3». На вкус owner'а — главное **видимые, с размером**, чтобы `gap` читался, а с `wrap` они переносились.
- Размер плиток подобрать так, чтобы в узком контейнере `wrap` визуально срабатывал (фикс ширина/мин-ширина у детей).
- Фон/бордер самого контейнера — опционально (можно оставить лёгкий, чтобы границы Flex читались), но **суть в детях**.
- Это JSON-схема пресета (`ISchema` nodes) — дети это node'ы с `type` из registry (`ui.Skeleton`/`ui.Card`/`ui.Typography`), `parentId` = id Flex-узла, и в `children` Flex-узла их id.

Если у Flex несколько пресетов (row/col/…) — каждый получает детей, чтобы его вариация была наглядна.

## Правка 2 — убрать белый бордер у Select-дропдауна

**Симптом:** content-панель выпадашки `Select` (Kobalte) на тёмной теме показывает **белый бордер** (виден, когда дропдаун открыт — отсюда «то появляется/исчезает»). Юзер: убрать.

- Найти в Select-компоненте (`packages/web/kit/ui/src/primitives/select/`) бордер на content/popover-панели.
- Убрать его **в корне** (не overlay-костыль): если бордер хардкодом/неправильным токеном (напр. белый вместо `border-border`) — убрать или привести к теме так, чтобы на тёмном фоне не было белой рамки.
- Проверить, что то же не торчит у родственных поповеров (Dropdown/Menu/Tooltip), если бордер из общего стиля — но менять только если это тот же источник; не расширять scope без нужды.
