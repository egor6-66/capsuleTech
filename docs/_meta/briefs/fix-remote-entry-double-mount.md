# Brief — `owner-builders`: fix double-mount в RemoteManifestPlugin

**Зона**: `packages/builders/vite/`
**Тип**: bug-fix, root-cause (не workaround)
**Scope**: один файл + его тест
**Branch**: `fix/remote-entry-no-double-mount`

---

## Контекст (что НЕ работает)

В iframe-режиме universal-canvas рендерится **дважды** — две копии "Capsule App"-страницы рядом в DOM внутри одного iframe. Direct `http://localhost:3000/` (standalone) показывает корректно — одну копию. Дубль появляется **только** через iframe boot-path (`<Remote.View name="universal-canvas">` host-side).

Verified в браузере 2026-06-22:
- iframe count в DOM = 1 ✅
- внутри iframe — две копии страницы (один root, два child-дерева)
- standalone :3000 = 1 копия

## Root cause (точное место)

**Файл**: `packages/builders/vite/src/plugins/remoteManifest.ts`
**Функция**: `buildUserAwareEntry()` (строки 82-103)

Текущий генерируемый `.capsule/remote-entry.ts` (когда у user есть `src/remote.ts`):

```ts
export const bootstrap: IRemoteBootstrap = (root, ctx) => {
  const dispose = createRoot(Bootstrap, { container: root });   // mount #1
  const userDispose =
    typeof (user as { bootstrap?: IRemoteBootstrap }).bootstrap === 'function'
      ? (user as { bootstrap: IRemoteBootstrap }).bootstrap(root, ctx)  // mount #2
      : undefined;
  return () => {
    if (typeof userDispose === 'function') userDispose();
    dispose();
  };
};
```

**Bug**: после Phase 1a finalize (#413) user'ский `src/remote.ts` экспортирует `bootstrap`, которая внутри вызывает `createCapsuleApp(root, {...})` — **полный mount** routeTree в `root`. Но генератор **до этого** уже сам мountит default `Bootstrap` в тот же `root` через `createRoot(Bootstrap, { container: root })`. Solid `render()` **аппендит**, не clearит — два дерева в одном root.

Generator был спроектирован под старый паттерн «user augments default mount». Новый канон (createCapsuleApp): **user.bootstrap = full mount, replaces default**.

## Что менять

В `buildUserAwareEntry()` поменять логику с «mount default + maybe-call-user» на **«call user if exists, else mount default»** (XOR):

```ts
function buildUserAwareEntry(): string {
  return [
    '// auto-generated by RemoteManifestPlugin -- do not edit.',
    "import type { IRemoteBootstrap } from '@capsuletech/web-remote';",
    "import { createRoot } from '@capsuletech/web-core/create';",
    "import { Bootstrap } from './bootstrap';",
    "import * as user from '../src/remote';",
    '',
    'export const bootstrap: IRemoteBootstrap = (root, ctx) => {',
    '  const userBootstrap = (user as { bootstrap?: IRemoteBootstrap }).bootstrap;',
    "  if (typeof userBootstrap === 'function') {",
    '    return userBootstrap(root, ctx);',
    '  }',
    '  return createRoot(Bootstrap, { container: root });',
    '};',
    '',
  ].join('\n');
}
```

**Контракт после фикса:**
- Нет `src/remote.ts` → `buildDefaultEntry()` (без изменений, mount default)
- Есть `src/remote.ts`, экспортирует `bootstrap` → вызвать только user'ский (он сам делает полный mount через createCapsuleApp)
- Есть `src/remote.ts`, НЕ экспортирует `bootstrap` → fallback к default (runtime branch)

`buildDefaultEntry()` (строки 68-80) **не трогать** — там mount default один раз, корректно.
