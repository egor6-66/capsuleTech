# Метамир: shell + модули, и «один модуль — два входа»

> [!abstract] Разворачиваем не «игру», а **платформу-метамир** (концепт GTA-RP-хаб) с
> подключаемыми **модулями** (shooter, racing, lang-lesson…). Ключевая идея интеграции:
> **один модуль запускается двумя способами** — внутри мира ИЛИ как отдельная веб-мини-аппа.
> Это ровно ваш «remote WebGL-модуль», только переиспользуемый.

## Видение и гардрейл
- **Север (north star):** живой open-world хаб в духе GTA-RP, куда встроены активности.
- **Гардрейл (чтобы не утонуть):** GTA-RP-масштаб — это годы. Строим **инкрементально**:
  сначала **shell + система модулей** (дёшево, максимальный рычаг), мир — грейбокс-район,
  мультиплеер — по вашим фазам 0-3 (не строим GTA-netcode сейчас). Первые модули — то, где
  уже есть разгон: **LangLesson** (переиспользует контент) + маленький **Shooter-sandbox**
  (переиспользует персонажа/оружие, которых уже делаем).

## Структура проекта (уже заложена в `game-unity`)
```
Assets/_Project/
  Core/            # SHELL, не зависит от модулей (asmdef CapsuleWorld.Core)
    Bootstrap/ Player/ Weapons/ Characters/ Camera
    Services/      # ContentApi, Auth(cookie), STT/TTS/LLM клиенты (origin-only)
    WebBridge/     # postMessage/jslib: init-конфиг ↓, события ↑
    ModuleSystem/  # IExperienceModule, ModuleContext, ExperienceRegistry
    UI/
  Modules/         # каждый модуль — свой asmdef, ссылается на Core, НЕ на другие модули
    LangLesson/    # CapsuleWorld.LangLesson  (реализует IExperienceModule)
    Shooter/  Racing/
```
- **asmdef-границы:** модули зависят от Core, но не друг от друга → чистые швы, быстрый
  компайл, честная модульность.
- **Контракт модуля** (уже написан): `IExperienceModule { Load(ctx); Enter(LaunchMode); Exit() }`,
  где `LaunchMode = EmbeddedInWorld | StandaloneWeb`. Один код — оба входа.

## ⭐ Главная развилка к вам: размер билдов (per-module vs монолит)
Стендэлон-мини-аппа (learn-апп рендерит ТОЛЬКО урок) НЕ должна тащить весь мир.
Предлагаю **две build-конфигурации из одного проекта**:

| Билд | Что внутри | Куда |
|---|---|---|
| **World build** | хаб + все модули (Addressables-стриминг сцен) | сам метамир |
| **Mini build** (per-module) | **Core-lite** (services/bridge/UI) + ОДИН модуль | встраивание в веб-апп (напр. lang-lesson в learn-апп) |

→ Из этого вытекает требование: **Core надо расщепить на `Core-lite` (сервисы/мост/UI,
без тяжёлого мира/персонажа) и `Core-world`** (контроллер, стриминг, netcode). Тогда
`lang-lesson` как мини-аппа весит мегабайты, а не сотню.

**Вопросы:**
1. Стендэлон-мини-аппа = тот же механизм remote-WebGL-модуля, что и остальной веб? Т.е.
   вы грузите per-module mini-build так же, как сейчас 3D-канвас?
2. Ок с двумя build-конфигами (World / Mini) и расщеплением Core-lite/Core-world?
3. **init-мост** должен уметь: `{ moduleId, launchMode, config }` — хост говорит, какой
   модуль и в каком режиме поднять. Это вписывается в ваш init-контракт?

> будут кейсы «встроить просто 3D-сцену или один объект». Цель: не дублировать код,
> механики не разъезжаются, **один флоу под разные кейсы**.

## Гранулярность встраивания (один флоу, разные юниты)
Встраиваемый юнит — спектр, всё через один Addressables-механизм:
**объект → сцена → модуль → мир**. Мелкий кейс (вставить 3D-объект/сцену в веб-страницу)
и крупный (полноценный модуль или хаб) — одна и та же система адресов/бандлов, разный
размер. Никакого параллельного кода: «вставить объект» = мини-подмножество того же флоу.

## Ленивая и прогрессивная загрузка (Addressables)
- **Lazy:** ассет/сцена грузится async по требованию, с плейсхолдером. Кэш бандлов в
  браузере (IndexedDB) → повторный заход мгновенный.
- **Progressive low→high:** показать LOW сразу → async-грузить HIGH → hot-swap. Тиры
  low/mid/high от арт-стороны = стрим-варианты.
- **Политика качества на юните** (метадата): `instant` (всегда low), `progressive`
  (low→high в фоне, дефолт), `quality-first` (сразу high + спиннер, для витрин).
- Текстуры — mipmap-стриминг Unity (низкие мипы первыми, по дистанции) — бесплатно.
- ⚠️ WebGL-память: HIGH-варианты выгружаем вне видимости, держим бюджет hero-hi.

## Сервисы shell'а (общие для веба и мира)
Через мост/по origin-API: **Content** (слова/дриллы/уроки по ссылке), **Auth** (cookie
едет сама), **STT/TTS/LLM** (для ролеплей-модуля). Все — только на ваш origin, конфиг
через init. Это общий слой и для мини-аппы, и для мира.

## Netcode (напоминание, не сейчас)
GTA-RP много-игроков = ваша фаза 3 (WebRTC DataChannel + self-hosted TURN + FishNet,
авторитет на вашем сервере). Фазы 0-2 (соло, turn-based) идут раньше и модулям хватает.
Мир на старте — **соло**, «RP» пока = окружение + активности, без live-мультиплеера.
