# ТЗ: «Finish» — эффект-мод поверхностей (хук в web-ui/lib, на существующих токенах)

> Автор: design-owner. Исполнение — owner-web-ui (фазы 1-2). Это спецификация, не реализация.

## 🎯 Цель {#goal}

Лёгкий **«финиш» поверхностей** как **эффект-мод поверх базового стиля**: градиент-поверхность + hairline-подсветка верхней кромки + мягкая глубина. Мод идёт **сверх** темы/палитры/(light·dark) и просто **добавляет эффекты** — палитру и скелет НЕ трогает. Референс — premium-карточки бенто.

## ❗ Чем это НЕ является (ловушки прошлых попыток — НЕ повторять)

- ❌ **Никакого хардкода в app/playground.** App уже использует кит — эффект придёт сам, когда мод активен.
- ❌ **Никаких новых токенов** (`--card-surface`/`--surface-highlight` и т.п.). Эффект лепится из **УЖЕ СУЩЕСТВУЮЩИХ** токенов (freeze-политика, ADR 042).
- ❌ **Не лезем в web-shell / app / mini-app / web-style** на этом шаге. Они юзают кит, который сам навешивает эффект.
- ❌ **Не на все компоненты сразу** — сперва ОДИН (`widget-frame`), потом тем же хуком на остальные.

## 🧩 Механизм — ОДИН переиспользуемый хук в `@capsuletech/web-ui/src/lib`

- Рядом с `createInfiniteScroll` / `createPagination` — новый хук (имя на усмотрение, напр. `createFinish` / `useFinish`).
- **Окутывает примитив:** когда финиш-мод активен (класс/`data-attr` на предке) — отдаёт surface-эффекты; когда не активен — **no-op** (примитив выглядит как сегодня).
- Эффекты **composed из существующих токенов** → тинт **автоматически следует теме** (одно определение работает на всех палитрах).
- Один источник в `lib` → дальше «надевается» на любой примитив без копирования.

## 🎯 Первый и единственный таргет сейчас — `web-ui/src/primitives/widget-frame`

Почему он: `WidgetFrame` оборачивает виджеты → эффект виден широко на ОДНОМ изменении. Конкретно:

- В `widget-frame.tsx` есть **surface-слой** `surfaceLayerStyle()` (≈стр. 132-139) с `background: var(--color-card)`. Это и есть точка приложения финиша.
- `widget-frame` вызывает хук из `lib` → при активном моде surface-слой получает: градиент-поверхность (вместо плоского `--color-card`), верхний hairline-инсет, мягкую глубину. При выключенном — откат на `var(--color-card)` = **текущий вид, ноль регресса**.
- Без новых props по возможности — через хук/стиль/классы.

## 🎨 Из чего лепим эффект (только существующие токены)

- **Surface-градиент:** `--card`/`--background` как база + `--foreground`/`--primary` с **низкой альфой** (верх чуть светлее низа). Через `color-mix(...)` или градиент-строку из этих токенов.
- **Hairline-кромка:** верхняя inset-линия из `--border`/`--foreground` low-alpha.
- **Глубина:** существующие `--shadow-*`.

Точные комбинации/альфы **подбираем живьём в браузере, правя сам хук** (`web-ui/lib`) — это пакет, HMR подхватывает. Тема для подбора — **deepPurple** (тинт ярче всего виден).

## 🪜 Фазы {#phases}

1. **Хук + widget-frame (текущий шаг, owner-web-ui ТОЛЬКО):** хук в `web-ui/lib` (на существующих токенах) + opt-in в `widget-frame` surface-слой. Активация — простой класс/`data-attr` (тоггл вручную для теста). НИ web-style, НИ app, НИ shell. Крутим значения в хуке вживую.
2. **Раскатка:** тот же хук на остальные примитивы (Card и др.) — без переписывания механизма.
3. **(Позже, отдельно) Формализация мода:** активацию финиша оформить как ось-мод в `@capsuletech/web-style/switcher` (поверх theme/density). Лесенка `flat | soft | glass` — **осознанная линейная лесенка, не две под-оси** (стекло без амбиента не приходит; `flat` независим). `flat` = буквальный no-op (мод выключен). Сюда же позже — амбиентный фон (app-shell слой). **В фазах 1-2 этого НЕТ.**

## ✅ Приёмка

- Мод выключен → вид **пиксель-в-пиксель как сегодня** (no-op).
- Мод включён на `widget-frame` → surface получает subtle градиент + верхний hairline + глубину; текст-контраст сохранён; «не датированно».
- **Ноль новых токенов. Ноль кода в app/shell/web-style** (фазы 1-2). Всё в `web-ui` (`lib` + `widget-frame`).
- Хук **переиспользуем** — «надевается» на другой примитив без копирования.
- Тинт следует теме (проверить deepPurple/zen/black). Перф: без `backdrop-blur`.
- `pnpm --filter @capsuletech/web-ui test` — green.

## ❓ Открытые (мелкие, owner-web-ui выберет)

1. Имя хука (`createFinish` / `useFinish` / `withFinish`).
2. Конвенция активации для теста (класс `.finish` / `data-finish` на предке). Формальная ось-мод в switcher — фаза 3, не сейчас.

## 🗺️ Делегирование

| Зона | Кто |
|---|---|
| Хук в `web-ui/lib` + opt-in на `widget-frame` (фаза 1) | owner-web-ui |
| Раскатка хука на остальные примитивы (фаза 2) | owner-web-ui |
| Ось-мод в switcher + амбиент-фон (фаза 3) | owner-web-style + design-owner |

## 📦 Статус реализации (2026-06-09)

**Сделано (в дереве, не закоммичено):**
- Хук `createFinish` (`web-ui/src/lib/finish/`) — градиент-поверхность + верхний hairline + цветная глубина из существующих токенов (`--color-foreground`/`--color-card`/`--color-primary`). Завайрен в `Card`, `widget-frame`, `Dropdown.Content`/`SubContent`, `Select.Content`. OFF = `{}` (истинный no-op).
- Switcher: `finishMode` стор в web-style (`useFinishMode`/`setFinishMode`/`toggleFinishMode`, ставит `data-finish` на `<html>`, дефолт OFF, localStorage). Контрол — `Shell.ModeToggle mode="finish"` (иконка `Sparkles`). В playground-хедере + превью на `/workspace/ui`.

**Подтверждённый вид (одобренный пресет):** topForeground 9% / mid card 70% @45% / bottom primary 18%; hairline 40%; inner-border 6%; contact `0 1px 2px rgb(0 0 0/.4)`; glow primary 22% `0 8px 24px`.

## 🔧 Продвинутый финиш (текущий трек)

**Баг (фиксим первым):** активация через `el.closest('[data-finish]')` на mount хрупка к таймингу прикрепления (портал/навигация) → финиш пропадает после реоткрытия дропдауна / навигации. **Фикс:** хук читает реактивный сигнал `useFinishMode()` из web-style напрямую (не DOM-walk); `finishGlobal.ts` (MutationObserver+closest) удаляется. `data-finish` остаётся как CSS-хук, но источник правды — сигнал.

**Параметры → реактивный стор `finishConfig` в web-style** (тип + стор живут в web-style — web-ui ниже зависит и читает; обратная зависимость = cycle, запрещена). Хук читает стор реактивно → живой подбор + редактор работают. Дефолты = одобренный пресет.

**Новые кнобы (берём сейчас):**
- **inner-only** (`innerOnly: boolean`) — тени только внутри (outer off, insets on). [референс: bento-скрин 1]
- **центр-свечение** (`centerGlowAlpha`, `centerGlowSize`) — radial-слой по центру, primary-тинт. [скрин 2]
- **прозрачность** (`surfaceAlpha` < 1) — полупрозрачная поверхность (красиво поверх амбиент-фона). [скрин 5]

**FinishEditor** (web-style/editor по образцу ThemeEditor + контрол в web-shell) — живые крутилки всех кнобов, открывается как «настройка палитры». **Freeze-safe:** кнобы = интенсивности/позиции/тоглы из существующих color-токенов, НЕ новые color-роли — палитра/Figma-контракт целы.

**Амбиент-фон** — параллельный трек: full-screen radial-свечения, тонированные `--primary` (зона app-shell/web-style). Подобранный инъекцией пресет: углы 16/13% + accent 8%, `background-attachment: fixed`.

## ⏸️ Отложено — «подумать» (НЕ выбрасывать)

- **Декор-фигуры/паттерны** (bento-скрин 4: концентрические кольца, замок-в-кольцах, графики, стопки карт) — это **контент-иллюстрации per-card, НЕ финиш поверхности**. Под общий мод не ложатся. Отдельная инициатива «фоновые SVG-паттерны/декор» (библиотека фигур). Большой самостоятельный разговор.
- **backdrop-blur (настоящее стекло)** — нужен позже; ТЗ флагует перф на повторяющихся карточках. Думаем КАК (opt-in для немногих элементов, не дефолт).
- **edge-fade/размытие краёв контента** (скрин 3) — `mask-image`-vignette дёшево, реальный блюр дорого. Отложено.
