# Handoff — 2026-06-05 (code-splitting + paused typing initiative)

Передача следующему агенту. Прочитай целиком — тут два трека: завершённый (ADR-034 code-splitting) и **паузнутый исходный** (типизация store/entity, ради него всё и начиналось).

## TL;DR {#tldr}

1. **ADR-034 (code-splitting) — СДЕЛАНО и в main.** Реестр слоёв переехал с runtime-`globalThis`-объекта на module-barrel + auto-import → бандлер делает route-scoping нативно. Verified (функционально в браузере + chunk-граф из артефактов).
2. **Паузнутый трек №1 (резюмить первым) — типизация `store.ctx.data` + entity-row.** Это был исходный запрос пользователя. План готов (variant B + `$infer`), producer-side уже есть (`fd39443`), осталась consumer-сторона. Детали ниже.
3. **App follow-ups (ewc)** — мелкие анти-паттерны/ошибки, НЕ CI-блокеры (см. почему ниже). Список ниже.

## 1. ADR-034 code-splitting — done

Доки: [[034-code-splitting-granularity|ADR 034]]. На ветке `feat/code-splitting-granularity` → смержено в main.

**Что сделано (owner-builders, `CapsuleRegistryPlugin`):**
- `.capsule/registry/` теперь **barrel-дерево** (тень папок `src/{widgets,views,…}`, `export * as` для вложенности, `export { default as X }` листья, `package.json` с `sideEffects:false`). Заменил объект `wrappers.ts`.
- `Object.assign(globalThis)` **убран**. Слои приходят через `unplugin-auto-import` (`import { Widgets } from '@capsule/registry'` инжектится; app-код без ручных импортов — DX цел).
- alias `@capsule/registry` регистрируется в **`config()`-хуке** плагина (НЕ `configResolved` — был баг: build работал, dev падал; фикс — commit `053119b`).
- Удалены legacy `wrappers.ts` + `slots.d.ts` (типы текут из барела через auto-import `capsule-imports.d.ts`).
- Тесты: `capsuleRegistry.test.ts` — 160 green.

**Доказанный результат (ewc prod-build):** 33 чанка, `index` 247kB (было 280). `login.chunk` импортит только `vendor-core·auth·index` — НЕ тянет dashboard-слои (`world`/`incidentsTable`/`incidents`); `dashboard.chunk` тянет ровно свои. Route-scoping нативный, per-layer waterfall устранён.

**Остаток ADR-034 (follow-up, не блокеры):**
- web-core: пустые `declare global { interface Widgets {} }` в `interfaces.ts` стали vestigial (никто не мерджит) — owner-web-core подчистит.
- `registry/packages.ts` (ADR 033 optional packages) использует тот же `globalThis`-паттерн — кандидат на такой же barrel-трактмент.
- heavy-leaf opt-in lazy-маркер (`// @capsule:lazy`) — пока не нужен (у ewc нет тяжёлого app-виджета; `dataTable` лэйзится сам из web-ui).
- проверить sandbox/другие apps на наличие старых `wrappers.ts`/`slots.d.ts` (плагин удалит их при первом запуске).

## 2. ⏸ ПАУЗНУТЫЙ ТРЕК — типизация store/entity (резюмить ПЕРВЫМ)

**Это был исходный запрос.** Сейчас в app-коде, чтобы типизировать `store.ctx.data` и тип строки entity, нужны `import` + `as`-каст (напр. [`apps/ewc/src/widgets/tables/incidents.tsx`](apps/ewc/src/widgets/tables/incidents.tsx) — `store?.ctx.data as IIncidentsContext`; [`apps/ewc/src/features/incidents.ts`](apps/ewc/src/features/incidents.ts) — `import { z }` + руками продублированный `interface IIncidentsContext`).

**План (согласован с пользователем). Зона owner-web-core, всё внутри web-core, web-state не трогаем:**

| # | Кусок | Файл | Суть |
|---|---|---|---|
| 1 | **Entity `$infer`** | `entity/types.ts` | враппер несёт phantom `$infer = z.infer<schema>` → `type IIncident = typeof Entities.Incident.$infer` (без импорта `zod`) |
| 2 | **Phantom-carry TCtx** | `interfaces.ts` | возврат `IControllerWrapper` += `& { readonly __ctx?: TCtx }` → `typeof Features.X` несёт контекст. **Сейчас стирается** ([interfaces.ts:589](packages/web/core/src/wrappers/interfaces.ts#L589) — возврат `(props)=>any`) |
| 3 | **`CtxOf<F>` + `StoreOf<F>`** | новый глобал | `CtxOf<F> = F extends {__ctx?: infer C} ? C : never`; `StoreOf<F>` = `IBridge` с `ctx.data: CtxOf<F>` |
| 4 | **Widget/Page `store` generic** | `interfaces.ts` + `widget.tsx` | `store`-арг типизируется из аннотации фабрики: `Widget((Ui, store: StoreOf<typeof Features.Incidents>) => …)` |

**Уже готово (`fd39443`, producer-side):** `IDefineStateSchema<TCtx>`/`IStateHandlers<TCtx>` — TCtx инферится из поля `context`, хендлеры Feature/Controller получают типизированный `context`. PoC-риск «выведется ли TCtx сквозь index-signature» — снят. `154737e` распаковывает `store.ctx.data` → `context` честен.

**Что осталось:** consumer-сторона (куски 1-4). Главный PoC-риск — #4 (инференс store из аннотации фабрики; контравариантность; fallback — явный генерик `Widget<typeof Features.X>`).

**App-миграция после framework-части (моя зона была):**
- Feature `incidents.ts`: удалить `interface IIncidentsContext` (дублирует `context`-блок, CtxOf его выводит) + `type IIncident = typeof Entities.Incident.$infer` (drop `zod`).
- Widget'ы (`tables/incidents`, `maps/world`, `sidebars/main`): `store: StoreOf<typeof Features.Incidents>` → drop import+каст.
- **settings-View'ы → Widget'ы** (см. follow-up ниже) — НЕ через хук. Идею `useStore<F>()` для View'ов **отвергли**: View не должен лезть в store (Решение A1). Контролы читают store → значит они Widget'ы (тонкие: store→props→stateless View).

## 3. App follow-ups (ewc) — реальные, но НЕ CI-блокеры

**Почему не блокеры:** CI Typecheck = `nx run-many -t typecheck`, но у apps target **отключён** (nx дизейблит из-за `noEmit:true` в project-references → просто echo, exit 0). Значит **CI вообще не тайпчекает apps/ewc** — эти ошибки слиплись на develop прямыми коммитами и CI их не ловит. Пакеты typecheck'аются и зелёные.

- **Dashboard page-loaders** ([`pages/workspace/dashboard/index.tsx`](apps/ewc/src/pages/workspace/dashboard/index.tsx)): `skeleton: <Ui.Skeleton …>` в 3 слотах — `Ui.Skeleton` не на `PageUi`. **Фикс (агреед, тривиальный, делать первым):** убрать `skeleton:` из слотов; `tables/incidents` и `maps/world` виджеты уже несут 2й-арг loader; `sidebars/main` — добавить `(Ui) => <Ui.Skeleton variant="card"/>` 2-м аргом. После barrel'а per-layer-lazy нет → slot-skeleton (chunk-load fallback) больше не нужен, остаётся только data-load loader виджета.
- **Cards page Typography** ([`pages/workspace/cards/index.tsx:26`](apps/ewc/src/pages/workspace/cards/index.tsx#L26)): `<Ui.Typography>` в Page (нет на `PageUi`). Фикс: вынести заголовок в тонкий View (ViewUi имеет Typography) или решить, нужен ли Typography в PageUi.
- **settings View→Widget** ([`views/settings/tableSync.tsx`](apps/ewc/src/views/settings/tableSync.tsx), `mapSync.tsx`): сейчас View лезут в `useCtx().store.ctx.data` — нарушение A1. Перевести в Widget'ы (`store: StoreOf<…>`), презентацию оставить stateless View'ом, кормить props. Dashboard-слот: `settings: <Widgets.Settings.TableSync/>`.
- **Compliance (page-слой импортит web-router):** `pages/workspace/{cards,reports,cards/[id]}/index.tsx` импортят `@capsuletech/web-router` — в слое `page` запрещено (allowed: `@tanstack/solid-router`). Режим `warn`, не валит build.

## 4. CI gap (инфра, для owner-builders/owner-tests)

**Apps не тайпчекаются в CI.** nx дизейблит `typecheck` target у проектов с `noEmit:true` в project-refs (все apps). Поэтому type-ошибки в apps/* проходят CI незамеченными (отсюда §3). Стоит починить (отдельный typecheck-target для apps, напр. `tsc --noEmit` без project-references-ловушки) — иначе apps копят type-долг.
