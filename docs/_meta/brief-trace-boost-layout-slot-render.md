# Brief — addendum: трейс РЕНДЕРА контента слота Matrix (owner @capsuletech/boost-layout)

**ADR:** [[062-runtime-observability-trace-channel]]. **Зона:** `packages/web/boost/layout/`. Аддендум к `e411e0bd` (там добавлен `slot:mount/dispose` — это **контейнер**).

## Зачем (охота bug A — дотрейсиваем недостающий узел)
Дамп трейсов: `remote.component:mount` ×2 (дубль канваса), но **узел-создатель не покрыт**. Мы трейсим контейнер слота (`slot:mount/dispose`), но **НЕ рендер КОНТЕНТА** — место, где Matrix выводит/инстанцирует `slot.children`. Канвас-виджет = `slots.main.children`; если Matrix выводит его **дважды** (measure-pass / dnd-ghost / два места рендера), получаем 2 виджета → 2 компонента. Этот узел надо затрейсить, чтобы увидеть дубль у источника (а не выпиливать Matrix).

## Что сделать
Добавить трейс в **КАЖДУЮ точку, где Matrix выводит/инстанцирует `slot.children`** (не контейнер, а сам контент):
```ts
trace('boost-layout.matrix.slot', 'render', { slot: <name> });
```
- Если контент слота рендерится в нескольких местах (скрытый measure-контейнер + видимый, draggable-обёртка + реальный, `<Dynamic>` дважды и т.п.) — трейс в каждом. **Смысл аддендума именно в этом:** увидеть, сколько раз инстанцируется контент конкретного слота.
- node `boost-layout.matrix.slot`, phase `render`, уровень `debug`. `trace` уже импортирован (из `e411e0bd`).

## Что покажет
В браузере (трейс по дефолту on): при заходе в студию для слота `main` —
- `slot:render` **дважды** → Matrix двоит контент слота (корень bug A здесь, копаем почему: measure/dnd/два места);
- **один раз** → Matrix контент рендерит однократно, дубль НЕ в Matrix → ищем в виджете (app-узел, параллельный трейс).
