# ADR 075 — Unity-метавселенная: языковой ролеплей

## Решение {#decision}

### D1 — Встройка {#d1}
Unity = **WebGL**, грузится remote-модулем (web-remote / canvas-host
`ICanvasEngineAdapter` — Unity в его платформо-листе изначально). Community —
портал/вход. Мост host↔Unity — версионируемый JSON-контракт (vendored,
прецедент канваса). Билд подчиняется ADR 072: все вызовы — только наш origin,
конфиг через init-мост, никаких зашитых внешних сервисов.

### D2 — Разделение и данные {#d2}
Unity: мир/лобби/аватары/механики/модели. Capsule: identity (кука),
контент-данные, голосовой стек, транспорт, рейтинг. **Сценарии — лесенка
сложности** (user): (1) ограниченный контекст — размеченные биты диалога с
дистрактор-сетами подсказок («4 из 5 уместны» → выше уровнем — наоборот);
(2) несколько тем вперемешку; (3) свободное общение — уместность судит LLM
на лету. Сеты = данные (авторство: учитель через vault + LLM-генерация под
judge, ADR 073); контент не дублируется — Unity потребитель наших API.

### D3 — Два судьи {#d3}
1. **Судья выбора** — уместность фразы: по разметке сета (уровни 1-2 лесенки,
   детерминированно) / LLM (свободный режим).
2. **Судья произношения** — юзер ОБЯЗАН произнести выбранное: голос-чанк →
   STT → совпадение с выбранной фразой (чекер-нормализация); фонемный
   скоринг произношения (wav2vec2-класс) — апгрейд позже.
**STT строится расширением `backend/voice`** (`/voice/transcribe`, Whisper;
py3.11+torch уже там, веса в кэше) — реализует ADR 065 ф.3; отдельный сервис
не заводим (voice = speech-capability в обе стороны). Это ближайший
пререквизит всего направления.

### D4 — Подсказки, рейтинг, анти-абьюз {#d4}
- **Уровень подсказок выбирает сам юзер** (свой опыт / что качает) —
  участники одной комнаты играют на разных сетах, каждому свой челлендж.
- **Очки — с хэндикапом**: вклад реплики = f(вердикты двух судей) ×
  множитель уровня подсказок; победитель — по нормированному счёту
  (новичок и опытный соревнуются честно). Точная формула — итеративно,
  принцип хэндикапа — канон.
- **Анти-абьюз by design** (user): оба судьи — рантайм (не самооценка),
  накрутка = реально говорить правильно = цель продукта. Спец-механик не
  строим до появления проблемы.
- События `roleplay.*` (сессия/реплики/итог) → event-журнал community
  (ADR 071) → рейтинг/прогресс в lang-профиле.

### D5 — Онлайн: единый флоу (правка user — принята) {#d5}
3D-лобби с аватарами — С ПЕРВОГО мультиплеер-релиза (наша сторона от этого
не меняется: комнаты/присутствие на нашем транспорте). State-sync
лобби-класса (ходьба/присутствие) толерантен к латентности — это НЕ
шутер-требования. **Netcode — только self-hosted** (Mirror / FishNet /
Netcode-for-GO с кастомным transport'ом — выбор за Unity-архитектором,
вопрос в онбординге; гейм-сервер хостим мы: VPS/нода ADR 072). Внешние
netcode/relay-облака (Photon, Unity Gaming Services) — исключены (ADR 072).
**Голос диалога — НЕ live-стрим**: реплика по очереди = чанк в наш STT тем
же путём, что из веб-аппа. Free-roam-шутер = та же инфраструктура с
повышенным tick-rate — включается, когда мир и аудитория дозреют.
Proximity-voice «просто тусить» — открытый вопрос к Unity-архитектору
(кандидат: WebRTC, отдельным решением).

### D6 — Канал {#d6}
Общение с Unity-архитектором — файлы `docs/_meta/unity/` через user
(liaison-канон учителя действует: инструкции с образцами, спека = onboarding).

## Фазы {#phases}
0. **STT** в voice (+ smoke «сказал→распознал→совпало»).
1. **Solo-NPC ролеплей** (STT+LLM+TTS, без сети) — первый играбельный релиз.
2. **Мультиплеер v1**: 3D-лобби (netcode self-hosted) + turn-based 1-на-1 +
   темы/уровни/подсказки + рейтинг.
3. Расширение миров: сценарии-интерактивы из идей-каталога (sandbox
   предлогов, ось времён, grabla-hunter...) + шутер-режим.

## Последствия {#consequences}
**Плюсы:** уникальная фича (говорение в игре) на 80% из готового стека;
контент один на веб и мир; рейтинг/identity переиспользованы; netcode-риск
изолирован от языкового ядра (turn-based не лагает by design). **Цена:**
STT-волна немедленно; гейм-сервер = новый инфра-компонент (ADR 070 ярусы);
зависимость от внешнего Unity-разработчика (митигируется контрактом моста и
каналом docs/_meta/unity).
