# ADR 072 — User-hosted nodes: рамки-констрейнты (дизайн потом)

## Контекст {#context}

GPU-peer из ADR 070 (локалка user'а с voice/image за WG-туннелем) — де-факто
ПЕРВАЯ user-hosted нода экосистемы. Обобщение до «любой юзер приносит свою
машину и опционально шарит её» требует в будущем: реестра нод и их
capabilities, node-scoped доверия, packaging'а «одна команда — нода поднята»,
модели пула (квоты/репутация — сшивается с community-рейтингами ADR 071).
Ничего из этого не строим преждевременно — но текущие решения не должны
этому противоречить.

## Констрейнты (обязательны для всех owner'ов с сегодняшнего дня) {#constraints}

1. **Никаких предположений о размещении.** Сервис не знает, на чьей машине он
   и где его зависимости: все ссылки на соседей — ТОЛЬКО env-URL. Хардкод
   host'ов/портов чужих сервисов в коде = канон-violation (это уже канон
   ADR 067/070 — здесь фиксируется его federation-причина).
2. **Внутренние каналы — за заменяемым швом аутентификации.** Shared-secret
   (`X-Internal-Key`, ADR 071 D4) — валидная v1 для «все сервисы наши», но
   контракт эндпоинта не должен зависеть от механизма: замена на node-scoped
   ключи/mTLS не меняет форму API. Не проектировать эндпоинты в расчёте на
   «доверенную сеть навсегда».
3. **Хранилище = S3-протокол, не файловая система.** Диск конкретной машины —
   не контракт. Любое новое хранение файлов — через S3-совместимый слой
   (MinIO сейчас; нода юзера с его хранилищем — потом, тем же протоколом).
4. **Каждый бэк-сервис контейнеризуем с первого дня.** Prod-форма = Docker
   (ADR 070); будущая «нода юзера» = тот же compose-файл у него. Сервис,
   который нельзя поднять контейнером с env-конфигом, — регрессия.
5. **Связность нод — исходящая ОТ ноды** (WG-паттерн ADR 070). Никогда не
   требуем от юзера белый IP, проброс портов или входящие соединения.

## Что осознанно НЕ решается здесь {#not-here}

Реестр нод и discovery; выдача/ревокация node-ключей (ляжет на identities-ось
ADR 068 — аддитивно); packaging для не-разработчика; модель общего пула
(квоты, модерация, репутация через event-журнал ADR 071); биллинг/лимиты.
Каждое — отдельный ADR, когда дойдём.

## Последствия {#consequences}

**Плюсы:** федерация станет эволюцией (нода = уже существующий контейнер +
будущий реестр), не революцией; сегодняшняя цена — ноль нового кода, только
дисциплина. **Цена:** пять правил в голове у каждого owner'а; ревью architect'а
проверяет их на каждом новом бэк-контракте.
