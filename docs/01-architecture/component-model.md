# 🧱 Component Model — сущности, списки, пресеты, слоты (БАЗА)

> [!important] Это база фреймворка, не «стайл-гайд»
> Всё ниже — фундаментальный флоу компонентов и композиций Capsule. Нарушение = не косметика, а поломка модели (studio-стор, рендерер, федерация датасетов). Читать ДО любой работы с UI — apps, packages, kit.

## Один закон
**Любая визуальная композиция живёт СТРОГО в `@capsuletech/web-ui`, пресетом. Никогда в пакете/аппе.** Пакет/апп **только**: выбирает пресет → кормит совместимые данные → гасит/оверрайдит слоты пропсами. Он **никогда не строит лайаут/вёрстку/механику**.

**Почему это база, а не вкусовщина:**
- **Рендерер (`@capsuletech/web-renderer`) обязан уметь рисовать ВСЁ.** Композиция вне кита он нарисовать не может.
- **Studio-стор** (хедлайнер) должен содержать всё, что где-либо используется. Композиция вне кита проходит **мимо стора** → для конструктора не существует.
- **Модульность:** пакеты не строят свои композиции → нет дрейфа, всё переиспользуемо между аппами.

Композиция вне кита = мимо рендерера И стора = **для фреймворка не существует**.

## Сущность (Card) vs Список (List)

### Карточка — это СУЩНОСТЬ
Карточка рисует **данные по ключам**. Внутри неё **ничего не строят руками** — ни Flex, ни ручных бейджей, ни лайаута.

- **Пресеты** = разные отрисовки сущности (фулл / компакт / медиа-сверху…). Разные пресеты = разный вид, одна сущность.
- **Слоты вкл/выкл** = полнота в рамках ОДНОГО пресета. «Где-то фулл-инфо слова, где-то урезано» — это **тот же пресет** с погашенными слотами, НЕ новая вёрстка.
- **Оверрайд пресета пропсами** — точечная гибкость в месте использования, без кардинальных изменений.
- **Мапер встроен в пресет** — ключи забиты в него. Урок / слово просто берёт подходящий пресет карточки и отдаёт данные. Всё.

> [!warning] Диагностика неправильного использования
> **Если внутри карточки где-то надо строить лайаут вручную — значит используем её не по назначению.** Ручной `Card + Flex + Typography + Badge` в пакете/аппе = антипаттерн. Правильно: `<Card preset="…" {…данные} slots={…} />`.

### Список — тот же принцип
- **Пресеты списка** = формы отрисовки (колонка / сетка / ячейки / …). Не «список одинаков везде» — у него **много пресетов**.
- **Элементы списка** = любые компоненты/композиции (в т.ч. карточка-сущность). Список не диктует, что внутри элемента.

## Данные — своя экосистема
Мы работаем в **своей экосистеме → наши данные совместимы с нашими компонентами.** Каждый пресет мапит **свои** ключи.

> [!important]
> **Если мы не можем нарисовать свои же данные своими же компонентами — мы делаем что-то неправильно.** Это сигнал ошибки в модели данных или в использовании компонента, не повод строить вёрстку руками.

**Будущее (адаптеры):** на каждый компонент — адаптер. Юзер через studio-интерфейс строит **мапер-адаптер**, подводя свои внешние датасеты под наши компоненты и инфру. Пресет + адаптер = любой датасет рисуется нашим компонентом.

## Регистрация (обязательна, два слоя)
Чтобы компонент реально был «в ките» (виден стору + рисуется рендерером):
1. **`packages/web/kit/ui/src/manifest/registry.ts`** `ALL[]` + `<comp>.manifest.tsx` (`type:'ui.X'`, `propsSchema`, `presets` = JSON `ISchema` → палитра рисует `<Renderer static>`). → видимость в **сторе/палитре**.
2. **`packages/web/runtime/core/src/ui-kit/imports.tsx`** — член `Ui.X` в namespace. → **registry рендерера** (`<Renderer registry={{ ui: Ui }}/>`). Гейт: `manifest-path-invariant.test.ts` требует `manifest.type` ⊆ `Ui` dot-path (оба слоя landing вместе, иначе CID красный).

Data-driven компонент → данные **plain/сериализуемые** (не accessor-функции), иначе preset (JSON ISchema) не собрать. Runtime-хендлеры (`onSelect`) и слоты-инъекции (`itemPreview` → кит сам оборачивает в Tooltip) в preset-схему не входят.

## Чек перед UI-работой
- [ ] Строю лайаут/композицию в пакете/аппе? → **СТОП.** Это пресет в ките.
- [ ] Карточка со слотами руками? → используем не по назначению, гаси слоты пресета.
- [ ] Новый вид? → пресет (или оверрайд пропсами), не новая вёрстка.
- [ ] Не рисуются свои данные? → ошибка в данных/использовании, не повод верстать.
- [ ] Компонент новый → зарегистрирован в ОБА слоя (manifest + Ui-namespace)?
